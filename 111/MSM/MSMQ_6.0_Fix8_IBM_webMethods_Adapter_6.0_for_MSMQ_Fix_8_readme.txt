7.0 Fix Contents

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

MMQ-132
Updated copyright notes, vendor, and product names for IBM rebranding.

MMQ-108 (MSMQ_6.0_Fix7)
In webMethods MSMQ Adapter, you can now add fatal error messages for the key 
fatalErrors in the property file. 

You can add fatal error messages for the fatalErrors key in msmqConfig.properties 
file using the following loaction: 
WmMSMQAdapter > config > msmqConfig.properties. 
Use semicolon (;) as separator to add multiple fatal error messages. 

This issue is now resolved. 

MMQ-100 (MSMQ_6.0_Fix6)
In webMethods MSMQ Adapter, an error occurs while trying to enable 
the listener once the MSMQ server restarts. 

When you try to enable the listener which listenes to a remote queue, 
the listener fails with the following error message: 
Cannot receive message from a transactional queue . The handle is 
invalid. 

This issue is now resolved.

MMQ-94 (MSMQ_6.0_Fix5)
In webMethods MSMQ Adapter 6.0, adapter services fails due to the 
broken connections. 

This issue occurs when MSMQ server is restarted and the 
adapter services to fail with the following error message: 
"The Message Queuing service was restarted. Any open queue 
handles should be closed. " 

This issue is resolved. Now, when you restart of the MSMQ server, 
the cononections in connection pool are refreshed, after the first 
failure of the adapter service using the connection

MMQ-75 (MSMQ_6.0_Fix4)
webMethods MSMQ Adapter does not support remote transactional 
read. 

The adapter cannot read messages from a remote queue on an 
MSMQ Server. 

The issue is resolved. The adapter now supports remote 
transactional read.

MMQ-70 (MSMQ_6.0_Fix2)
Cannot configure adapter connections to MSMQ Server 4.0 or 5.0 
when using webMethods MSMQ Adapter 6.0. 

When configuring an adapter connection, the user needs to select 
a version of MSMQ Server to connect to, choosing a value from the 
MSMQ Version field. However, there is no value for MSMQ Server 
4.0 or 5.0. 

This issue is resolved. webMethods MSMQ Adapter 6.0 is now 
supported for use with MSMQ Server 4.0 and 5.0. Specifically, the 
MSMQ Version field on the adapter's Configure Connection Type 
screen now contains values for MSMQ Server 4.0 and 5.0. 

MMQ-10 (was 1-1GMRRW) (MSMQ_6.0_Fix1)
On a periodic basis the MSMQ Adapter fails to peek/get messages 
from the queue.

After the Integration Server is restarted, with an interval 
varying from 8 to 18 days, the MSMQ Adapter fails to peek/get 
messages from the queue even when the adapter is enabled. The 
MSMQ Adapter throws the following error messages:
- Cannot peek message contents. (null). Error Code = 80020009
- Cannot peek message contents. Not enough storage is available
  to complete this operation. Error Code = 80020009

This issue is now resolved.
