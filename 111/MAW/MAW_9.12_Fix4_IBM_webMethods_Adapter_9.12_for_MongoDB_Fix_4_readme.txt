7.0 Fix Contents
In addition to including resolutions to product defects, a fix may also include 
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

MAW-124
Updated copyright notes, vendor, and product names for IBM rebranding.

MAW-94  (MAW_9.12_Fix3)
In webMethods Adapter for MongoDB, the Test Connection in Integration Server
Administrator fails with an error as Integration Server is unable to create
the connection

This issue is now resolved.

MAW-90  (MAW_9.12_Fix3)
In webMethods Adapter for MongoDB, a connection cannot be created if X.509
certificate is used for authentication.

This issue is now resolved.

MAW-69  (MAW_9.12_Fix2)
webMethods Adapter for MongoDB now supports MongoDB Atlas.

MAW-38 (MAW_9.12_Fix1)
In webMethods Adapter for MongoDB, the filter in the select service is
incorrectly created if the field is an array of embedded documents.

This issue is resolved.

MAW-36 (MAW_9.12_Fix1)
In webMethods Adapter for MongoDB, the select service is not listing a field
in the SELECT tab and WHERE tab(filter), if the field value is null in the
sampling documents.
This issue is resolved.

MAW-33 (MAW_9.12_Fix1)
In webMethods Adapter for MongoDB, if a field has multiple data type values in 
MongoDB collections, the Select service fails with below error: 

Error occurred while selecting document. "Unsupported value for output 
parameter"<field name>. 

This issue is resolved now. A new Output Filed Type, 'java.lang.Object', along 
with other types will appear in the list.
