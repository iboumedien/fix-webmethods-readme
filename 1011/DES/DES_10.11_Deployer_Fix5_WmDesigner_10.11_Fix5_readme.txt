1.0 Fix Name

WmDesigner 10.11 Fix 5 for Deployer



2.0 Fix ID

DES_Deployer_10.11_Fix4



3.0 Product(s)/Component(s) Affected

This fix affects Designer Process Development for Deployer (WmDesigner package).



4.0 Requirements and Dependencies


For language pack requirements, see the Globalization section of the Software AG Empower Product 
Support website at https://empower.softwareag.com/Products/Globalization/default.asp



5.0 Platform Support

This fix is supported on the same platforms as webMethods Designer Process Development 
Deployer 10.11 (WmDesigner package).



6.0 Cautions and Warnings

This fix needs to be installed on all servers hosting the WmDesigner package, including 
the Deployer source and target servers.



7.0 Fix Contents


WMDPD-10431
If you configure watt.server.http.X-Content-Type-Options= to "nosniff", the WmDesigner package page does not function properly.

The issue is resolved.

WMDPD-10352
webMethods Process Development uses an older version of the Apache Commons IO library, 
which has known security vulnerabilities.
Process Development now uses Apache Commons IO 2.11.0.

The issue is resolved. 

WMDPD-10298(DES_10.11_Deployer_Fix3)
In webMethods Process Development, when you install WmDesigner_10.7_Fix6 and start 
Integration Server, an error message appears in the 
server log: "Missing class wm.designer.sim.fitDistribution."

The issue is resolved.

WMDPD-10244(DES_10.11_Deployer_Fix2)
In webMethods Deployer, when you roll back an initial deployment to the target server, 
the process models remain enabled for execution.

The issue is resolved. The process models are now disabled after the roll back.

Important - If the project defines deployment sets that include Integration Server assets, 
they are removed from the target server after the roll back. This can lead to issues if any 
process instances exist for the given process models.

WMDPD-10151(DES_10.11_Deployer_Fix1)
In Process Deployer build of BPM deployment set will fail when the set contains task assets.

This issue is resolved.

WMDPD-10200(DES_10.11_Deployer_Fix1)
With webMethods Process Development, checkpoint generation fails during asset deployment 
if the Integration Server host uses a PostgreSQL database.

The issue is now resolved.



8.0 Installation

This fix should appear in Update Manager under Integration Server > WmDesigner Fix

Use Command Central to install fixes on multiple machines from a central  location (9.7 or later 
products), or Software AG Update Manager to install fixes on one machine at a time. For instructions,
see the documentation at http://documentation.softwareag.com.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action 
will revert your installation to the previously installed fix. You cannot use these instructions to 
uninstall the previously installed fix.

Uninstall using Command Central (9.7 or later) or Software AG Update Manager. For instructions, see 
the documentation at http://documentation.softwareag.com.



10.0 Globalization

This fix conforms to the internationalization standards of the webMethods product suite and includes 
support for operation in any country, locale, or language as specified in the Installing webMethods 
Products guide. It was not tested with non-English configurations and non-ASCII data. However, this 
fix has no globalization impact and can be applied to systems running in any supported locale or 
configuration.



10.1 Localization

This fix does not require an updated language pack. It might contain new messages and these messages 
will appear in English.
