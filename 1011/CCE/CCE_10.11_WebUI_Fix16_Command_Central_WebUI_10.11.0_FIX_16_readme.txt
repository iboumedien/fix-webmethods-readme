7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before applying the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

CCE-8385
In the Command Central web user interface, when you navigate to Installations 
and attempt to sort the available installations by status, the Status column 
does not display the installations in the correct order.

The issue is resolved. 

CCE-8371
In the Command Central web user interface, the Create Mirror Repository dialog 
box does not display the last letter of long repository names in the 
"Repositories to use as sources" and "Product repositories to filter by" fields.

The issue is resolved. 


CCE-8209
In the Command Central web user interface, when you navigate to Installations 
> Environments, if the list of environments does not fit on the screen and
you scroll down to select an environment from the bottom of the list, 
the scroll bar incorrectly goes back to the top.

The issue is resolved. Now the scroll bar stays at the correct position.

CCE-8193
In the Command Central web user interface, the Create Mirror Repository dialog 
box does not fully display long repository names in the "Repositories to use as 
sources" and "Product repositories to filter by" fields.

The issue is resolved. Now a scroll bar appears on the bottom of the fields if 
they contain repositories with long names. 


CCE-7828
When creating a runtime or database layer in a stack, if you want to re-use the 
value of a property from other layers in the stack, the list of available properties 
is empty.
The issue is now resolved. 


CCE-7944
In the Stacks view of the Command Central web user interface, when adding a new 
node with a new runtime and a new database layer, the tooltip and labels for the 
available properties for the database layer are incorrect. Furthermore, the dropdown 
list for selecting property values from another layer is empty but the button for it 
appears enabled.

The issue is resolved. The toolttip and labels are now correct and the button for empty 
dropdown list is disabled. 

CCE-8007
In the Stacks view of the Command Central web user interface, when trying to add 
a new runtime layer to an existing node in a stack, the "Add layer" wizard freezes 
on the "Identify target nodes" section and does not allow you to select an existing node. 
The issue occurs when the stack contains three or more nodes.

The issue is resolved. You can now add a new runtime layer to an existing node in 
a stack that contains three or more nodes. 


CCE-7964
The Command Central web user interface stops responding when you navigate to 
CCE > Command Central Server in the local installation, select the Configuration 
tab, and click the CCE link to navigate back.

The issue is resolved.


CCE-7569
With Command Central, the logs in the Jobs view might not properly display 
several of the last log entries.

The issue is resolved. The window toolbar no longer displaces the last log entries.


CCE-7342
Command Central fails with an error when a new remote installation is added from the Web 
user interface using SSH Private Key connectivity credentials.

When executing a remote Platform Manager bootstrap using SSH private key credentials, 
the execution fails with an error that the provided private key does not exist.

The issue is resolved. Platform Manager is now installed successfully on a remote machine 
when using SSH private key credentials.


CCE-7158
Attempting to view the contents of a repository using the Command Central web user interface 
fails with the following error: "There is no repository to register". The issue occurs when 
the repository name contains a space.

The issue is resolved. 


CCE-7151
When using the Templates view in the Command Central Web User Interface, any selected custom 
sorting order is reset after deleting a composite template.

The issue is resolved. Custom sorting orders are no longer reset after deleting a composite template.


CCE-7156
In the Command Central web user interface, when auto-creating stacks or creating stacks from scratch, 
the wizard does not list all possible releases in the "Product release" drop-down box.

The issue is resolved. Now Command Central provides all possible product releases when creating 
stacks.


CCE-7199
When upgrading stacks in Command Central, the values in lists, such as Bootstrap Installer, 
Product repository, Fix repository, and Credentials, are not listed in any particular order.

The issue is resolved.


CCE-7244
When adding an installation using the Add Installation wizard in Command Central, the values in lists, 
such as Bootstrap Installer and Credentails, are not listed in any particular order.

The issue is resolved. The values in the Bootstrap Installer list are sorted by 
release and fix number. The values in the Credentials list are sorted alphabetically.



8.0 Installation

Use Command Central to install fixes on multiple machines from a central 
location (9.7 or later products), or Software AG Update Manager to install 
fixes on one machine at a time. For instructions, see the documentation at
http://documentation.softwareag.com.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently 
installed fix. This action will revert your installation to the previously 
installed fix. You cannot apply these instructions to the previously 
installed fix.

Uninstall using Command Central (9.7 or later) or Software AG Update Manager. 
For instructions, see the documentation at
http://documentation.softwareag.com.
