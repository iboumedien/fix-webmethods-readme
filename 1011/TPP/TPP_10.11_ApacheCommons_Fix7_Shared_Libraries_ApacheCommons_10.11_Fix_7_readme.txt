1.0 Fix Name

webMethods Shared Libraries ApacheCommons 10.11 Fix 7



2.0 Fix ID

TPP_10.11_ApacheCommons_Fix7



3.0 Product(s)/Component(s) Affected

This fix affects the common extended libraries installed with the 10.11 webMethods products.



4.0 Requirements and Dependencies

Fixes are cumulative. Therefore, for each of the listed fixes, you can install the specified fix or 
later.



5.0 Platform Support

Same platforms as product release.



6.0 Cautions and Warnings

None.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

TPS-4535 (TPP_10.11_ApacheCommons_Fix7)

webMethods Shared Libraries ApacheCommons 10.11 Fix 6 contained incorrect fix dependencies.

The issue is resolved. The fix dependencies have been changed.

TPS-4449 (TPP_10.11_ApacheCommons_Fix6)

webMethods Shared Libraries contains the third party Apache Commons-IO with version 2.11.0, which has known security vulnerabilities.

The issue is resolved. The following library has been updated:
The Apache Commons-IO library is updated from version 2.11.0 to version 2.15.1
The Apache Commons-lang3 is updated from v3.12.0 to v3.14.0
The Apache Commons-codec is updated from v1.15.0 to v1.16.1

TPS-3417 (TPP_10.11_ApacheCommons_Fix5)

webMethods Shared Libraries contains the third party Apache Commons Configuration with version 2.8.0, which has known security vulnerabilities.

The issue is resolved. The following library has been updated:
The Apache Commons Configuration library is updated from version 2.8.0 to version 2.10.1

TPS-2254 (TPP_10.11_ApacheCommons_Fix4)

webMethods Shared Libraries contains the third party Apache Commons Codec with version 1.13, which has known security vulnerabilities.

The issue is resolved. The following library has been updated:
The Apache Commons Codec library is updated from version 1.13 to version 1.15

TPS-2111 (TPP_10.11_ApacheCommons_Fix3)

webMethods Shared Libraries contains the third party commons-fileupload with version 1.4, which has known security vulnerabilities:
[CVE-2023-24998].

The issue is resolved. The following library has been updated:
> commons-fileupload is updated from v1.4 to v1.5
> commons-io is updated from v2.10.0 to v2.11.0

TPS-1959 (TPP_10.11_ApacheCommons_Fix2)

webMethods Shared Libraries contains commons-text with v1.8.0 which has a critical security vulnerability associated with it [CVE-2022-42889].

The issue has been resolved. The following libraries have been updated:
> commons-text is updated from v1.8.0 to v1.10.0
> commons-lang3 is updated from v3.9.0 to v3.12.0

TPS-1835 (TPP_10.11_ApacheCommons_Fix1)

WebMethods Shared Libraries contains commons-configuration2.jar v2.7.0 which has a critical security vulnerabilities associated with it.

The issue is resolved commons-configuration2.jar has been upgraded to v2.8.0
