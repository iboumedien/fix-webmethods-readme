End to End Monitoring Integration Server or Microservices Runtime Plug-in 10.11 Fix 12

3.0 Product(s)/Component(s) Affected

This fix affects
webMethods Integration Server 10.11

7.0 Fix Contents


UHM-7496 (UHM_10.11_ISOrMSRPlugin_Fix12)

End to End Monitoring not supported E2EM On-Premise agent trace redirection to some APM servers like (AppDynamics, Dynatrace, New Relic, Elastic APM, IBM Instana, etc.,)
 
This is resolved now. Self-hosted Transaction Monitoring with OpenTelemetry will support all the APMs that have a native OTLP support for receiving a trace.
Description



UHM-7497 (UHM_10.11_ISOrMSRPlugin_Fix12)

End to End Monitoring not supported E2EM On-Premise agent trace redirection to some APM servers.

This is resolved now. Self-hosted Transaction Monitoring with OpenTelemetry will support few APM tools that support Trace ID length more than 16-byte and have a native OTLP support for receiving a trace.



UHM-4250 (UHM_10.11_ISOrMSRPlugin_Fix11)

Dynamic collector end point resolution was not working in onprem API gateway agent.

This issue is now resolved, collector end point gets resolved based on the tenant details provide in Integration server tenant connection.



UHA-36 (UHM_10.11_ISOrMSRPlugin_Fix10)

End to End Monitoring hybrid configurations were causing Integration server to go down

This is now resolved by updating Jackson Dependencies



UHM-3494 (UHM_10.11_ISOrMSRPlugin_Fix9)

End to End Monitoring Hybrid agent to collector connection was not happening when 2 way SSL was enabled in Integration Server.

This issue is now resolved.



UHM-3299 (UHM_10.11_ISOrMSRPlugin_Fix8)

End to End Monitoring used to only support monitoring of SOAP 1.1 API Gateway APIs. SOAP 1.2 monitoring was not supported.

This issue is nor resolved. End to End monitoring now provided support for monitoring both SOAP 1.1 and 1.2 APIs.



UHM-3298 (UHM_10.11_ISOrMSRPlugin_Fix8)

With log4j usage in End to End monitoring Integration server logs used to contain improper field values.

This issue is now resolved, End to End Monitoring now uses logback for logging support.



UHM-3138 (UHM_10.11_ISOrMSRPlugin_Fix7)

End to End Monitoring used to support just wmcloud authentication, there was no support for JNDI authentication.

This issue is now resolved, End to End Monitoring now supported both wmCloud and JNDI authentication.



UHM-3107 (UHM_10.11_ISOrMSRPlugin_Fix6)

For Hybrid monitoring to work collector endpoint needed to be manually added in the agent.config.

Now this issue is resolved, Collector endpoint is dynamically fetched from the tenant connection details provided in 
Integration server.



UHM-2831 (UHM_10.11_ISOrMSRPlugin_Fix5)

End-to-End Monitoring does not monitor the on premises Integration Server's Java Message Services(JMS), 
as a part of hybrid monitoring.

This issue is resolved. JMS messaging services are now monitored by End-to-End Monitoring.



UHM-2837 (UHM_10.11_ISOrMSRPlugin_Fix5)

CustomId was not displaying in case of hybrid monitoring use cases.

This issue is resolved



UHM-2548 (UHM_10.11_ISOrMSRPlugin_Fix2)

End-to-End Monitoring is unable to monitor transactions in Microsoft Azure tenants.

This issue is resolved. End-to-End Monitoring now supports multiple tenant connections along with 
Microsoft Azure tenant transactions.



UHM-2479 (UHM_10.11_ISOrMSRPlugin_Fix1)

End-to-End monitoring fails when Integration Server is enabled with multiple tenant connections.

This issue is resolved. End-to-End monitoring now successfully sends data to cloud for multiple 
tenant connections in the same region.



8.0 Installation

Install using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.

1. If this is the first time you are installing a fix using the Software AG Update Manager, 
   back up the End-to-End Monitoring file system before applying the fix.   

2. After fix installation, copy \E2EMonitoring\agent\plugins\uhm-onpremise-is-plugin.jar to
   \IntegrationServer\lib\jars directory.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action will 
revert your installation to the previously installed fix. You cannot apply these instructions to the 
previously installed fix.

Uninstall using the Software AG Update Manager. For instructions, see the documentation 
at http://documentation.softwareag.com.

After fix uninstallation, To revert the Software AG product installation file system to the state before fix installation,
delete changes made to the Software AG file system, along with this jar file \IntegrationServer\lib\jars\uhm-onpremise-is-plugin.jar
 


10.0 Globalization 

This fix conforms to the internationalization standards of the webMethods product suite and includes 
support for operation in any country, locale, or language as specified in the Installing webMethods 
Products guide. It was not tested with non-English configurations and non-ASCII data. However, this 
fix has no globalization impact and can be applied to systems running in any supported locale or configuration.
