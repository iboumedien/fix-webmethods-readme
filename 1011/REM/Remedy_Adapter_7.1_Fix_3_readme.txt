You must add the new watt property, watt.adapter.Remedy.includeDisplayFields and set it to true so that the fields are displayed in the adapter service template. REM-93 (RMDY_7.1_Fix1)In webMethods Remedy Adapter, the Get service fails with a java.lang.IndexOutOfBoundsException exception. The Remedy Adapter Get service fails with a java.lang.IndexOutOfBoundsException exception when you assign custom values to the ID Enumeration field for a Select Field type in an AR System form. This issue is resolved. Now, the Remedy Adapter Get service successfully obtains the values of the selected fields when the ID Enumeration field of a Select Field type is set to "Custom".5.0 Platform SupportThis fix is supported on all the platforms supported bywebMethods Remedy Adapter 7.1.6.0 InstallationInstall the fix using the Software AG Update Manager.For instructions, see Using the Software AG Update Managerlocated either in the _documentation directory or on thedocumentation Web site at http://documentation.softwareag.com.7.0 UninstallationUninstall the fix using the Software AG Update Manager.For instructions, see Using the Software AG Update Manager.NOTE: This uninstall procedure can only be used to uninstall themost recently installed fix. This action will revert yourinstallation to the previously installed fix. You cannot applythis uninstall procedure to the previously installed fix.8.0 Cautions and WarningsNone.9.0 GlobalizationThis fix conforms to the internationalization standards of thewebMethods product suite and includes support for operation inany country, locale, or language as specified in the webMethodsInstallation Guide. It was tested with non-English configurationsand non-ASCII data.However, this fix has no globalization impactand can be applied to systems running in any supported locale orconfiguration.9.1 LocalizationThis fix does not require an updated Language Pack.
