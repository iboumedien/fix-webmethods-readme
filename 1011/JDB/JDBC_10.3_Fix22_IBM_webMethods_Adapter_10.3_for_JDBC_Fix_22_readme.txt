2.0 Fix ID

JDBC_10.3_Fix22

None.



4.0 Requirements and Dependencies

This fix requires the installation of the latest Integration Server Core fix.

Fixes are cumulative. Therefore, for each of the listed fixes, you can install 
the specified fix or later.



5.0 Platform Support

Same platforms as product release.



6.0 Cautions and Warnings

None.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include 
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the 
release.

JDB-4177
While connecting to a PostgreSQL database using mTLS from JDBC, the default 
PostgreSQL SSL settings were getting applied and the user was not able to set 
sslmode property for mTLS. Changes have been made and now default PostgreSQL SSL
settings are only applied if the user has not already provided their own.

seconds that the connection pool checks for the connectivity of the idle connection.

JDB-3853 (JDBC_10.3_Fix19)
In webMethods Adapter for JDBC, names for notification resources such as buffer
table, sequence, and trigger cannot have custom prefixes.

This issue is now resolved. webMethods Adapter for JDBC provides the
"watt.adapter.JDBC.notification.useBaseNameAsPrefix" parameter to customize the
names of notification resources.

Set "watt.adapter.JDBC.notification.useBaseNameAsPrefix" to true to use the
Base Name of a notification as a prefix for notification resources such as
buffer table, sequence and trigger. The parameter applies only to SAP HANA.

JDB-3386 (JDBC_10.3_Fix15)

PostgreSQL database if the table names, column names, and procedure names are
defined in lower case in the database but are defined in upper case or camel case
in the service templates.

This issue is now resolved.

Set the postgresql.casesensitive connection property to true or false in the
Other Properties field. Possible values are:

quotes if the values are in upper case or camel case.

JDB-3374 (JDBC_10.3_Fix14)
In webMethods Adapter for JDBC, stored procedure services that contain a field
of JDBC Type BOOLEAN and input field of type java.lang.String, are unable to
process 0 and 1 as boolean value false and true respectively.

This issue is now resolved.

Set the watt property watt.adapter.JDBC.setBooleanAsBitString to true.
The watt property watt.adapter.JDBC.setBooleanAsBitString enables the adapter to
process 0 and 1 as boolean value false and true respectively in the following
conditions:
1. In stored procedure adapter services where JDBC Type is BOOLEAN, and Input
Field Type is java.lang.String
2. In adapter services where JDBC Type is BIT, and Input Field Type or Output
Field Type is java.lang.String

JDB-3372 (JDBC_10.3_Fix14)
A new field "Driver Group" is added to Connection Properties for JDBC Connection,
and JDBC SSL Connection. JDBC Connections can be configured to use different
versions of the same JDBC driver by selecting the appropriate Driver Group.

The field Driver Group, lists all the folders in Integration Server_directory/
instances/<instance_name>/WmJDBCAdapter/code/jars.
The Default Driver Group represents the JDBC driver JARs located in the folder
Integration Server_directory/instances/<instance_name>/WmJDBCAdapter/code/jars.
You must create a new folder in Integration Server_directory/instances/
<instance_name>/WmJDBCAdapter/code/jars and copy the JDBC driver JARs to this
folder, to create a new Driver Group.

JDB-2637 (JDBC_10.3_Fix5)
If connection pooling is enabled and when the when the minimum pool size is more 
than zero, the connection pool checks the connectivity of the connections that 
are idle for the value of Heart Beat Interval. If the idle connection is stale 
then the connection pool is reset.

The value for Heart Beat Interval is in seconds. 
The default value for Heart Beat Interval is zero

For example, if the value of Heart Beat Interval is 25 seconds, 
connection pool looks for connection that are idle for 25 seconds. 
If the connection is stale then the connection pool is reset
