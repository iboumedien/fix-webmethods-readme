1.0 Fix Name

Metering Server 10.11 Fix 4



2.0 Fix ID

MTA_10.11_Server_Fix4


3.0 Product(s)/Component(s) Affected


This fix affects wM Metering 10.11.



4.0 Requirements

This fix requires Metering Server 10.11. 
Metering Server 10.11 Fix 4 
must be installed together with OSGI.Platform_10.11.0_Fix7 
if the products for these fixes are installed on the system. 

Official fixes are cumulative. Therefore, you can install the specified fix or later.

For language pack requirements, see the Globalization section of the Software AG
Empower Product Support website at
https://empower.softwareag.com/Products/Globalization/default.asp



5.0 Platform Support

This fix is supported on all platforms supported
by webMethods Product Suite 10.11.



6.0 Cautions and Warnings

Software AG Tomcat Server is stopped by Software AG Update 
Manager, therefore it must be restarted after applying the fix.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before applying the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

MTA-853
The Software AG Metering Server uses the third-party Liquibase library v4.4.3,
which has а security vulnerability CVE-2022-0839.
This issue is resolved. The Metering Server now uses the stable version
Liquibase 4.19.0.

MTA-958
The Software AG Metering Server uses the third-party Apache PDFBox library v.2.0.16,
which has а security vulnerability.
The issue is now resolved. The Software AG Metering Server no longer uses Apache PDFBox library.

MTA-523
The webMethods Metering Server uses basic authentication that does not allow you
to log out from the application.
This issue is resolved. The webMethods Metering Server now uses form-based authentication.

MTA-527
The date range picker in the Admin UI of the Software AG Metering Server
incorrectly adds two days to the selected end date.
This issue is resolved.

MTA-687
The Software AG Metering Server uses the third-party Gson library v2.8.5,
which has ? security vulnerability CVE-2022-25647.
This issue is resolved. The Metering Server now uses the stable version
Gson 2.9.0.

MTA-824
In the Software AG Metering Server UI, when you save a report, the file name might be truncated.
This issue is resolved.

MTA-825
The date range pickers in the Software AG Metering Server UI might display
incorrect day of week labels.
This issue is resolved.

MTA-926
The Aggregations section in the webMethods Metering Server Admin UI is obsolete.
The issue is resolved. The Aggregations section is now removed.

MTA-929
On the Usage Report page of the Software AG Metering UI, the Customer and Tenant
drop-down arrows appear double.
This issue is resolved.

MTA-631
After uninstalling a webMethods Metering fix, the webMethods Metering Server might not start.
This issue is resolved.

MTA-384
Software AG Runtime migration fails with an error: "Custom Wrapper Configuration
Migrator: Platform Migrator for Tanuki Wrapper Custom Configuration Migration failed."
This issue is resolved.
