7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before applying the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

OGI-4640
Software AG Security Infrastructure uses the third-party Bouncy Castle
library v. 1.76, which has known vulnerabilities.
This issue is resolved. Security Infrastructure now uses the stable version
Bouncy Castle 1.78.1

SIN-1628
In Software AG Security Infrastructure, you cannot enable Web SSO to use SHA-256 signatures and digests for sending requests to IdPs.
This issue is resolved. To enable Web SSO to use SHA-256, modify the system property com.softwareag.security.opensaml.default.digest=http://www.w3.org/2001/04/xmlenc##sha256 or the environment variable OPENSAML_DEFAULT_DIGEST=http://www.w3.org/2001/04/xmlenc#sha256. To modify default signature algorithm you can specify it the following way com.softwareag.security.opensaml.default.sign=RSA_http://www.w3.org/2001/04/xmldsig-more##rsa-sha256 or OPENSAML_DEFAULT_SIGN=RSA_http://www.w3.org/2001/04/xmldsig-more#rsa-sha256.

OGI-3941
The Software AG Security Infrastructure uses ESAPI 2.4.0.0 that has known vulnerabilities.
This issue is resolved. ESAPI is upgraded to the stable version 2.5.2.0.

OGI-3851
Software AG Security Infrastructure uses the third-party Apache Velocity Engine v.1.5,
which has a known vulnerability CVE-2020-13936.
This issue is resolved. Security Infrastructure now uses the stable version
Apache Velocity 2.3.

OGI-3880
Software AG Security Infrastructure uses the third-party Bouncy Castle library v. 1.69.0.04,
which has known vulnerabilities.
This issue is resolved. Security Infrastructure now uses the stable version Bouncy Castle 1.76.

OGI-3807
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities CVE-2023-34035.
This issue is resolved. Spring Security is upgraded to the stable
version 5.8.6.

SIN-1564
When you use the SAMLAssertValidatorLoginModule, the abort and logout messages
have an INFO log level instead of a DEBUG log level.

This issue is resolved.

OGI-3713
Software AG Security Infrastructure uses the third-party Spring Security framework v.5.7.5,
which has security vulnerabilities.
This issue is resolved. Security Infrastructure now uses the stable version
Spring Security 5.8.3.

OGI-3487
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities.
This issue is resolved. Spring Security is upgraded to the stable
version 5.7.5.

SIN-1487
Software AG Security Infrastructure uses the third-party Spring-security library which has а security vulnerability.
This issue is resolved. 

SIN-1481
The Software AG Security Infrastructure uses a version of
ESAPI that has known vulnerabilities.
This issue is resolved. ESAPI is upgraded to the stable
version 2.4.0.0. 

SIN-1461
Software AG Security Infrastructure uses the third-party XmlSec library v.1.5.6, which has а security vulnerability.
This issue is resolved.

SIN-1417
Upgrade of Spring Security to 5.5.2 for security reasons.

SIN-1436
In some cases if an attribute contains non UTF-8 character, signature validation of the SAML assertion might fail.

Issue is now resolved.
 

SIN-1411
In some cases keystore and truststore information for LDAPS connection was not properly loaded and secure connection could fail.

Issue is now resolved

SIN-1376
When checking password of LDAP user when using technical user connection to LDAP server, resources of the connection could be left to be released later.
Now all resources are released immediately.

The issue is now resolved.

SIN-1381
Added property to enable LDAP Login Module to log authentication errors at ERROR level

SIN-1394
If username contains LDAP special symbol and affixes are used authentication to LDAP might fail due to missing escape of the username.

The issue is now resolved.

SIN-1366
The Internal User Repository Command Line tool allows creating a user
with an empty name or password.
This issue is resolved.
