1.0 Fix Name

webMethods Monitor 10.11 MWS Fix 5 (Monitor portlets update)



2.0 Fix ID

MON_10.11_MWS_Fix5



3.0 Product(s)/Component(s) Affected

This fix affects webMethods Monitor 10.11.



4.0 Requirements and Dependencies

- DES_10.11_Process_Fix2

- Monitor 10.11.0 Fix 6 or newer. (WmMonitor package update)

- Optimize 10.11.0 Fix 1 or newer, even if Optimize is not present on your
system. This is because the Optimize fixes contain updates to shared components
used by Monitor as well.

- It is recommended to install the latest fixes for related BPM products
such as Designer, Process Engine, Optimize, ESB, and Business Console.



5.0 Platform Support

This fix is supported on all platforms supported by webMethods Monitor.



6.0 Cautions and Warnings

None.



7.0 Fix Contents

WMN-8464
In the Process Instances page of Monitor, you cannot limit 
the advanced search results to a specific date range.

The issue is resolved.

WMN-8419(MON_10.11_MWS_Fix4)

In the Process Instances page of Monitor, when you open a Child Instances For Step window, the table width might 
exceed the window size. The issue occurs when the Custom Process ID of a child process instance is longer than 48 
characters.
 
The issue is resolved.

WMN-8392(MON_10.11_MWS_Fix4)
In webMethods Monitor, when you go to the Process Instances page and open a Child Instances for Step window, 
the window is empty. The issue occurs when a process step has several child process instances.

The issue is resolved.

WMN-8235(MON_10.11_MWS_Fix4)
webMethods Monitor uses an older version of the jquery-ui.js and jquery-ui.min.js libraries, which have known security 
vulnerabilities. The issue is resolved. webMethods Monitor now uses jquery-ui.js 1.13.2 and jquery-ui.min.js 1.13.2.

WMN-8224(MON_10.11_MWS_Fix3)
In the Process Instances page of Monitor, when you perform an advanced search 
with a custom logged field, an error is displayed.

The issue is resolved. 
Install DES_10.11_Process_Fix2 and MON_10.11_MWS_Fix3, and redeploy your process models.

Note:- The Date format for logged Field Filter search value should be in the format
'YYYY-MM-DD' or 'YYYY-MM-DD HH24:MM:SS'

WMN-8217(MON_10.11_MWS_Fix2)
webMethods Monitor uses an older version of the Gson library,
which has known security vulnerabilities.

The issue is resolved. webMethods Monitor now uses Gson 2.10.1.


WMN-8125(MON_10.11_MWS_Fix1)

In webMethods Monitor, when you successfully start the audit data archive procedure,
the Archive Audit Page might incorrectly display an error: "An error was encountered
when starting archiving for your input criteria. Please verify that you have setup your
archive database and JDBC pool correctly.�

The issue is resolved.



8.0 Installation

Use Command Central to install fixes on multiple machines from a central  location (9.7 or later
products), or Software AG Update Manager to install fixes on one machine at a time. For instructions,
see the documentation at http://documentation.softwareag.com.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action
will revert your installation to the previously installed fix. You cannot use these instructions to
uninstall the previously installed fix.

Uninstall using Command Central (9.7 or later) or Software AG Update Manager. For instructions, see
the documentation at http://documentation.softwareag.com.



10.0 Globalization

This fix conforms to the internationalization standards of the webMethods product suite and includes
support for operation in any country, locale, or language as specified in the Installing webMethods
Products guide. It was not tested with non-English configurations and non-ASCII data. However, this
fix has no globalization impact and can be applied to systems running in any supported locale or
configuration.



10.1 Localization

This fix does not require an updated language pack. It might contain new messages and these messages
will appear in English.
