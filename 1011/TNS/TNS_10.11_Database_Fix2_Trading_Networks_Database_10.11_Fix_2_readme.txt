Trading Networks Database 10.11 Fix 2.

TNS_10.11_Database_Fix2

This fix affects Trading Networks Database 10.11
scripts

Trading Networks Server 10.11 Fix 2 or the latest fix that supersedes this.



5.0 Platform/RDBMS Support

Same platforms and RDBMS as product release.



6.0 Cautions and Warnings

None.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes 
for the release.

OTN-22854
In Trading Networks, TN_RUNTIMETABLES_DELETE takes longer to execute.
This issue is resolved. Now, the performance of this procedure is improved.

OTN-22611(TNS_10.11_Fix1)
TradingNetworks table creation failed with Amazon Aurora MySQL

OTN-22647(TNS_10.11_Fix1)
TradingNetworks table drop failed with Amazon Aurora MySQL

OTN-22754(TNS_10.11_Fix1)
In trading networks, the host field value truncates when this field has more
than 64 characters in Delivery Settings for a partner.
This issue is resolved. Now, the host field has a maximum of 512 characters.

8.0 Installation

Install using Software AG Update Manager. For instructions, see the 
documentation at http://documentation.softwareag.com.

1. Shut down all product instances that connect to the database and then 
   back up the database.

2. On the machine that hosts the Database Component Configurator, go to the 
   <Software AG_directory>/common/db/bin directory and run the command below.
   See Installing Software AG Products for command details.

   dbConfigurator.{bat|sh} -a migrate -d <RDBMS> -c <db_component[,...]> 
   -v latest -l <RDBMS_URL> -u <user_name> -p <password>

<db_component> names are: 
TradingNetworks, TradingNetworksArchive



9.0 Uninstallation

Uninstall using Software AG Update Manager. For instructions, see the 
documentation at http://documentation.softwareag.com.

If necessary, restore the database backup you created before migrating the 
database components.
