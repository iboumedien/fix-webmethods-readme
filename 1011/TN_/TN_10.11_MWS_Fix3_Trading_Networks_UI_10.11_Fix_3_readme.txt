7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes 
for the release.

OTN-23038
When you submit a larger XML file on the submit document page, the Mywebmethods 
server fails and displays the error message "400: Unable to parse form content" 
in the UI.
This issue is resoved.

OTN-23138
In webMethods Trading Networks, after installing the Japanese language pack the
TN transactions "Query or Transform failed" and the partners are recognized as
'Unknown'

OTN-23017
When My webMethods receives the xml document with the CDATA section. Xml CDATA 
makes My webMethods unresponsive.
This issue is resolved

OTN-22955 (TN_10.11_MWS_Fix2)
The error message "Can't find resource for bundle
com.webmethods.portal.framework.portletcontainer.portletapi.impl.PortletResourceBundle,
key error.message.26022" appears on clicking the "Add Partner Group" button or
when editing an existing partner group in Partner group.
This issue is now resolved.

OTN-22642 (TN_10.11_MWS_Fix1)
In webMethods Trading Networks, while creating a public search using the filter
option, you select as a default search (options tab >Basic as a default search
tab). Then, when you open the transaction page, it opens on the advanced tab,
not the basic tab and switching to basic clean the content and return to ?All.?
This issue is now resolved.
