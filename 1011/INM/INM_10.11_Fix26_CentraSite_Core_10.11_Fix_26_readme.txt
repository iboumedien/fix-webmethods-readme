In addition to including resolutions to product defects, a fix may also  include
enhancements to existing functionality and features.  Any  of  these  items  can
result in new or changed built-in services, APIs, or  configuration  parameters.
Review the fix contents carefully before applying the fix.

This fix is cumulative, which means that it is an aggregate  of  previous  fixes
for the release.

INM-25885
Set the HTTP header "Referrer-Policy" to "No-Referrer".

The Referrer-Policy header actively controls the inclusion of information about
the referring URL in the HTTP headers when a user navigates from one website to
another. This helps in protecting user's privacy and security.

Therefore, the HTTP header "Referrer-Policy" is set to "No-Referrer".

The issue is resolved.

INM-25717
The application does not validate for File Type or Content-Type in the avatar
field

Server-side validation is added for the file upload functionality in CentraSite.
The file upload fails if the file size is above the set limit(default 100 MB)
or the file is a binary (with mime type "application/octet-stream").

The maximum file size limit can be set in one of the config.xml files:
<CentraSiteInstall_Directory>/cast/cswebapps/BusinessUI/custom/conf/
centrasite.xml
<CentraSiteInstall_Directory>/cast/cswebapps/BusinessUI/system/conf/
centrasite.xml

Add the <FileUploadSettings maxUploadfileSize="" /> inside the UIProperties
tag.

For example:

<GUIConfiguration>
    ...
    <UIProperties>
        ...

        <FileUploadSettings maxUploadfileSize="104857600" />

        ...
    </UIProperties>
    ...
</GUIConfiguration>

The issue is resolved.

INM-25587
Vulnerable third party component Log4j is used

Severe vulnerabilities affecting multiple versions (2.0 to 2.17.0) of the Apache
Log4j 2 utility have been publicly disclosed. These vulnerabilities allow
unauthenticated remote code execution.

The issue is resolved by upgrading Log4j to version 2.17.1.


INM-25587
Vulnerable third party component Log4j is used

Highly severe vulnerabilities impacting multiple versions (2.0 to 2.16.0) of the
Apache Log4j 2 utility was disclosed publicly through the project's GitHub in
December 2021. This vulnerability allows for unauthenticated remote code
execution.

The issue is resolved by upgrading Log4j to version 2.17.0.


INM-25587
Vulnerable third party component Log4j is used

A highly severe vulnerability (CVE-2021-44228), impacting multiple versions (2.0
to 2.14.1) of the Apache Log4j 2 utility was disclosed publicly through the
project's GitHub on December 9, 2021. This vulnerability allows for
unauthenticated remote code execution.

The issue is resolved.

8.0 Installation

IBM strongly recommends you to take a backup of the internal database  in  which
CentraSite's registry and repository data is stored before installing this fix.

Install  using  IBM  webMethods  Update  Manager.  For  instructions,  see   the
documentation at https://docs.webmethods.io/.


8.1 Post-Installation Steps

It may be required  to  start  CentraSite  Runtime  and/or  CentraSite  Registry
Repository after fix installation.

Clear browser cache before starting CentraSite BusinessUI.



9.0 Uninstallation

You cannot uninstall the fixes applied on CentraSite 10.11.
