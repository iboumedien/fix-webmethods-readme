3.0 Product(s)/Component(s) Affected

4.0 Requirements and Dependencies

This fix requires Common Landscape Asset Registry 10.11. 
Common Landscape Asset Registry 10.11 Fix 9 
must be installed together with DesignerCPP_10.11.0_Fix2 
if the products for these fixes are installed on the system. 

Fixes are cumulative. Therefore, for each of the listed fixes, you can install
the specified fix or later.

For language pack requirements, see the Globalization section of the Software AG Empower Product
Support website at https://empower.softwareag.com/Products/Globalization/default.asp

Same platforms as product release.



6.0 Cautions and Warnings

All product profiles that run on top of OSGi runtimes are stopped by
the Software AG Update Manager and will automatically be started
after the session completes.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.


LAR-962
Software AG Common Landscape Asset Registry uses the third-party Eclipse Equinox CM
library v.4.25 that has known security vulnerabilities.
This issue is resolved. Common Landscape Asset Registry now uses the stable version
Eclipse Equinox CM v.4.30.

LAR-677
In Software AG Designer, when you use Deploy to Cloud to migrate a package in
webMethods Cloud Container, the migration fails with a promotion exception error.
This issue is resolved. An additional dependency evaluation is now performed.

LAR-625
Common Landscape Asset Registry log files contain exposed plain text credentials.
This issue is resolved. User credentials are no longer exposed in the log files.

LAR-657
Software AG Common Landscape Asset Registry uses a version of Apache Log4j 2
that has known vulnerabilities.
This issue is resolved. Software AG Common Landscape Asset Registry now uses
the stable version Apache Log4j 2.17.1.



8.0 Installation

Use Command Central to install fixes on multiple machines from a central 
location (9.7 or later products), or Software AG Update Manager to install 
fixes on one machine at a time. For instructions, see the documentation at
http://documentation.softwareag.com.

9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently 
installed fix. This action will revert your installation to the previously 
installed fix. You cannot use these instructions to uninstall the previously 
installed fix.

Uninstall using Software AG Update Manager. For instructions, see the 
documentation at http://documentation.softwareag.com.
