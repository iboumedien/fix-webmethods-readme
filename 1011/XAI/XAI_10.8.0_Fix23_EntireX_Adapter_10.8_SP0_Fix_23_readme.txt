7.0   Fix Contents

- EXX-15698 XAI_10.8_Fix23:
   IDL Extractor for webMethods IS does not recognize "minItems" and 
   "maxItems" for Open API array definitions.
   This issue is resolved now.


- EXX-15713 XAI_10.8_Fix23:
   CICS Socket Listener: Internal protocol mismatch during handshake 
   disconnects from host, sends data to the disconnected host (CICS 
   call failed: java.net.SocketException: Socket closed), reconnects 
   to the host and sends the data again.
   The issue is solved now and the data will be send with the lower 
   protocol in the same connection. 


- EXX-15822 XAI_10.8_Fix23:
   IDL Extractor for webMethods IS: The IDL file does not respect 
   the string length definitions in OpenAPI based REST Consumers.
   This issue is resolved now.






7.1   Resolutions included from earlier Fixes

Fix 19:

- EXX-15314 XAI_10.8_Fix19:
   Using openAPI-3 REST services containing a Date / Time field may 
   fail with 
   com.softwareag.is.rest.openapi.exception.InvalidDataFormatExceptio
   on
   The issue is now resolved.


- EXX-15431 XAI_10.8_Fix19:
   EntireX Adapter 10.8 Fix 15 is installed.
   However, the wmentirex.log however reports 10.8.0.1.963, Date: 24 
   May 2023 
   The issue is now resolved


Fix 15:

- EXX-15203 XAI_10.8_Fix15:
   Marshalling array data in groups to RPC can fail with 
   ArrayStoreException, depending on the Integration Server 
   pipeline.
   This issue is resolved now.


- EXX-15262 XAI_10.8_Fix15:
   RPC Call to EntireX Adapter Listener may fail with 
   IllegalArgumentException.
   This issue is now resolved.


- EXX-15213 XAI_10.8_Fix15:
   EntireX Adapter: group elements marshalled to object array 
   instead of mapping their value to child elements.
   This may result in errors like 08000500 reason: 
   java.lang.IllegalArgumentException
   This issue is resolved now.


Fix 13:

- EXX-15178 XAI_10.8_Fix13:
   Certain data type structures, such as arrays, were not marshalled 
   correctly and produced an exception 
   (java.lang.ArrayStoreException).
   This issue is now resolved. 


Fix 11:

- EXX-14892 XAI_10.8_Fix11:
   When suppressing null values of type date (D), setting 
   entirex.marshal.date=null resulted in an error.
   Also, when suppressing null values of numeric/decimal type (I1, 
   I2, I4, N, P) using the suppression mode "Suppress null elements 
   and trim array cells at end containing null elements and zero 
   numbers", zeros in structures/groups or as simple elements were 
   not suppressed.
   These issues are now resolved.


Fix 9:

- EXX-14091 XAI_10.8_Fix9:
   When trying to retrieve a list of objects via EntireX RPC 
   Listener connections from a REST API Descriptor, error 0800 0310 
   was thrown.
   This issue is resolved now.


- EXX-14648 XAI_10.8_Fix9:
   EntireX Adapter: Copy and Paste of Broker ID from and into 
   connections containing double back slashes will replace them at 
   runtime with double forward slashes.  This may cause errors like 
   "Wrong argument to Broker constructor: Invalid Broker ID"
   This issue is resolved now.



- EXX-14734 XAI_10.8_Fix9:
   EntireX Adapter CICS ECI Connection: The service (issuing the 
   CICS ECI call) does not allow to overwrite the user ID. 
   This could cause a Security response code from CICS as the call 
   could be issued with a invalid userid / password combination
   This issue is resolved now.


- EXX-14730 XAI_10.8_Fix9:
   Passwords are printed to the console in encrypted form.
   With this correction the (encrypted) password is no longer 
   displayed


- EXX-13802 XAI_10.8_Fix9:
   EntireX Application Monitoring Data Collector
   entirex.appmondc.filesperday=yes is specified.
   The new log file was started with a delay and data of the new day 
   was included in the log file of the previous day. No data from 
   the next day should be included in the file for the previous day.

   This issue is resolved now. 


Fix 3:

- EXX-13948 XAI_10.8_Fix3:
   The structure of the payload provided by the user exit for CICS 
   Socket Listener does not match the IDL structure. 
   The issue is solved now by rearrange the payload provided by the 
   user exit.
   
   With this correction the new payload format is:
   * Name (16 bytes, for example Commarea, LargeBuffer or the name 
   of a container)
   * Data-length (4 bytes)
   * Data (number of bytes defined in the previous parameter)
   
   Channel container data may contain multiple blocks (one for each 
   input container)


- EXX-14251 XAI_10.8_Fix3:
   A call to a listener connection with at least one output 
   parameter of type Date leads into the error : EXX 0800 0310 
   "Unsupported field: YearOfEra".
   The issue is now resolved


- EXX-14346 XAI_10.8_Fix3:
   url-style BrokerID starting with uppercase protocol (like SSL://, 
   HTTP:// etc.) was not accepted by validation process. The 
   application hit the error "Invalid Broker ID".
   This is resolved now.
   With this correction the protocol part of url-style BrokerID can 
   be written now case-insensitive.
