7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

LAR-1191

Common Landscape Asset Registry uses a version of Eclipse JGit v. 6.9.0
that has known security vulnerabilities - NVD - CVE-2025-4949.
This issue is resolved.
Software AG Common Landscape Asset Registry now uses the stable version 7.2.1.

LAR-976
Software AG Common Landscape Asset Registry uses the third-party engine
Google Code Gson 2.9.0, which has security vulnerabilities.
The issue is resolved. Software AG Common Landscape Asset Registry now uses Google Code Gson 2.10.1.

LAR-1107
In some rare cases assets might not be listed in the solution after solution update.
The issue is resolved. Now assets should always be listed.

LAR-827
Software AG Common Landscape Asset Registry uses the third-party Eclipse Equinox CM
library v.4.25 that has known security vulnerabilities.
This issue is resolved. Common Landscape Asset Registry now uses the stable version
Eclipse Equinox CM v.4.30.

LAR-804
Software AG Common Landscape Asset Registry uses a version of Eclipse JGit
that has known security vulnerabilities. This issue is resolved.
Software AG Common Landscape Asset Registry now uses the stable version 6.7.0.

LAR-805
Common Landscape Asset Registry log files contain exposed plain text credentials.
This issue is resolved. User credentials are no longer exposed in the log files.

LAR-802
In Common Landscape Asset Registry, when a composite asset and a regular asset contain
a property with the same name, the same value might be assigned to both properties.
This issue is resolved. A different value can now be assigned to each property.
