7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include 
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the 
release.

BDDR-202
Updated the Product name as per IBM rebranding and also Licenses 3rdparty 
notices for fix JDC_9.0_Fix5.

BDDR-88 (JDC_9.0_Fix4)
Upgraded Apache Cassandra Progress driver to 6.0.0.000464 version for
compatibility with Java 17.

BDDR-42 (JDC_9.0_Fix3)
The BigData drivers are in the locked state while you access the database objects.
This issue occurs only when the current applicationKey does not match existing 
applicationKey.

This issue is now resolved.


RGHADPESTBL-5221 (JDC_9.0_Fix2)
The driver jar file cassandra-db-6.0.0.jar is renamed to wmcassandra.jar.

The upgrade is from from 6.0.0.000118 (C0185.F000032.U000008) to 6.0.0.000216 
(C0285.F000105.U000052).


BDDR-2 (JDC_9.0_Fix1)
In webMethods BigData Driver for Apache Cassandra, using the existing 
cassandra-db-6.0.0.jar, Adapter for JDBC was unable to establish connection with 
Cassandra version 3.7. 

This issue is resolved.Now, cassandra-db-6.0.0.jar is upgraded to the latest 
build provided by Progress Data Direct.

Note: The name of the driver jar will be wmcassandra.jar.
The driver version is upgraded from 6.0.0.000118 (C0185.F000032.U000008) to 
6.0.0.000195 (C0253.F000060.U000026).
