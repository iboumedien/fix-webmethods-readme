1.0 Fix Name

webMethods Task Engine 10.15 Fix 4



2.0 Fix ID

TE_10.15_TE_Fix4



3.0 Product(s)/Component(s) Affected

This fix affects:
- My webMethods Server 10.15
- Task Engine 10.15
- Integration server 10.15



4.0 Requirements and Dependencies


Fixes are cumulative. Therefore, for each of the listed fixes, you can install the specified fix or 
later.

For language pack requirements, see the Globalization section of the Software AG Empower Product 
Support website at https://empower.softwareag.com/Products/Globalization/default.asp



5.0 Platform Support

Same platforms as product release.



6.0 Cautions and Warnings

None.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

MWS-37207 (TE_10.15_TE_Fix4)
Logging for CDS & TE inside of IS was different. CDS did not create any logging files and ended up
logging wherever Log4j was setup. Where as TaskEngine@IS created it's own te.log.

To streamline both product's logging inside of IS a common _full_.log(similar to MWS) is
introduced inside the Integration Server's logging directory.




WTN-5966 (TE_10.15_TE_Fix4)
The fix for WTN-5527 caused this customer's task rule changes to revert themselves after rule
execution.

The issue has been resolved.

MWS-31946 (TE_10.15_TE_Fix3)
A security issue with the third-party library jQuery was discovered in Task Engine on Integration
Server.

The issue is resolved. The jQuery library is updated to the more secure version 3.6.3.




MWS-32771 (TE_10.15_TE_Fix3)
A security issue with the third-party component moment.js was discovered in the WmCDS and
WmTaskEngine packages for Integration Server.

The issue is resolved.




MWS-32825 (TE_10.15_TE_Fix3)
In certain conditions the searchTasks built-in service for Task Engine might log a
java.rmi.UnmarshalException: java.io.IOException: java.lang.IllegalArgumentException:
java.lang.ClassCastException error message.

The issue is resolved.




MWS-33178 (TE_10.15_TE_Fix3)
Task Engine might fail to apply round-robin distribution for initial task assignments. This issue
occurs for LDAP users that have not logged in to the hosting My webMethods Server.

The issue is resolved.




MWS-33188 (TE_10.15_TE_Fix3)
Task Engine might fail to parse e-mails and perform automated actions for custom task
applications. This issue occurs when Task Engine runs on My webMethods Server.

The issue is resolved.




MWS-33242 (TE_10.15_TE_Fix3)
A security issue with the third-party library Eclipse Parsson was discovered in My webMethods
Server.

The issue is resolved.




MWS-33384 (TE_10.15_TE_Fix3)
Task Engine might recreate the task database tables during startup. This issue occurs when a fix
that includes changes to database tables is installed, even if Task Engine is disabled.

The issue is resolved.




MWS-31965 (TE_10.15_TE_Fix2)
In certain conditions a task instance might fail to transition to status Completed and report back
its status to the parent process.

This issue is resolved.




MWS-32576 (TE_10.15_TE_Fix2)
Task Audit view in My webMethods might fail to display audit information for some task instances.

This issue occurs on an installation that uses an Oracle database and has a large number of task
instances.

The issue is resolved.




WTN-5818 (TE_10.15_TE_Fix2)
Task Engine on Integration Server might log errors about missing adminapi.json file for the
WmTaskEngine package during initialization.

The issue is resolved.




WTN-5819 (TE_10.15_TE_Fix2)
Deploying task applications might fail when Task Engine is running on an Integration Server
instance, installed on a Linux operating system.

The issue is resolved.




WTN-5849 (TE_10.15_TE_Fix2)
The wmTaskEngine package in Integration Server might attempt to initialize some of the task
services and log initialization errors. This issue occurs when the wmTaskEngine services are not
explicitly enabled.

The issue is resolved. Now the wmTaskEngine package does not attempt to initialize unless you
explicitly enable the Task Engine core services by setting the -Dtask.library.mode.enable
property in the custom_wrapper.conf to true. Configure the property only if you plan to install
and use Task Engine on Integration Server.




WTN-5853 (TE_10.15_TE_Fix2)
Task Engine on Integration Server might fail to queue task instances. This issue occurs when you
attempt to start Task Engine before running all required database scripts, and prepare the
database subsequently and restart Task Engine.

The issue is resolved.




MWS-31864 (TE_10.15_TE_Fix1)
A security issue with the third-party component Elasticsearch was discovered in My webMethods
Server.

The issue is resolved. The Elasticsearch component is upgraded to a more secure version.




MWS-31951 (TE_10.15_TE_Fix1)
A security issue with the third-party library JSF 2.2.10 was discovered in My webMethods Server.

The issue is resolved. The library is updated to the more secure version 2.3.9 with an additional
customization that fixes CVE-2019-17091.








8.0 Installation

Install using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action 
will revert your installation to the previously installed fix. You cannot use these instructions to 
uninstall the previously installed fix.

You can only uninstall one Task Engine fix at a time. Make sure you backup your server settings 
before uninstalling Task Engine fixes.



10.0 Globalization

This fix conforms to the internationalization standards of the webMethods product suite and includes 
support for operation in any country, locale, or language as specified in the Installing webMethods 
Products guide. It was not tested with non-English configurations and non-ASCII data. However, this 
fix has no globalization impact and can be applied to systems running in any supported locale or 
configuration.



10.1 Localization


This fix does not require an updated language pack. It might contain new messages and these messages 
will appear in English.
