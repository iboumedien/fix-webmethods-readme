1.0 Fix Name

WmDesigner 10.15 Fix 5 for Deployer



2.0 Fix ID

DES_Deployer_10.15_Fix5



3.0 Product(s)/Component(s) Affected

This fix affects Designer Process Development for Deployer (WmDesigner package).



4.0 Requirements and Dependencies

None



5.0 Platform Support

Same platforms as product release.



6.0 Cautions and Warnings

This fix needs to be installed on all servers hosting the WmDesigner package, including 
the Deployer source and target servers.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

WMDPD-10835
With WmDesigner, adding or removing one or more logical server entries may hang and fail.

This issue is resolved.

WMDPD-10812(DES_10.15_Deployer_Fix4)
The WmDesigner package uses an older version of the Apache Commons IO library, 
which has known security vulnerabilities.

The issue is resolved. WmDesigner now uses Apache Commons IO 2.18.0.

WMDPD-10428(DES_10.15_Deployer_Fix3)
If you configure watt.server.http.X-Content-Type-Options= to "nosniff", 
the WmDesigner package page does not function properly.

The issue is resolved.

WMDPD-10352(DES_10.15_Deployer_Fix2)
webMethods Process Development uses an older version of the Apache Commons IO library, 
which has known security vulnerabilities.

The issue is resolved. Process Development now uses Apache Commons IO 2.11.0.

WMDPD-10298(DES_10.15_Deployer_Fix1)
In webMethods Process Development, when you start Integration Server, an error message 
appears in the server log: "Missing class wm.designer.sim.fitDistribution."

The issue is resolved.



8.0 Installation

Use IBM webMethods Command Central to install fixes on multiple machines from a central  location (9.7 or later 
products), or IBM webMethods Update Manager to install fixes on one machine at a time. For instructions,
see the documentation at https://docs.webmethods.io.
