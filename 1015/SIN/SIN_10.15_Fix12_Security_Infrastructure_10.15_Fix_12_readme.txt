1.0 Fix Name

Software AG Security Infrastructure 10.15 Fix 12



2.0 Fix ID

SIN_10.15_Fix12


3.0 Product(s)/Component(s) Affected


This fix affects IBM webMethods Security Infrastructure 10.15.



4.0 Requirements

This fix requires any of the products listed in 3.0 Products (s)/Component(s) Affected. 

Fixes are cumulative. Therefore, for each of the listed fixes, you can install
the specified fix or later.


5.0 Platform Support

Same platforms as product release.


6.0 Cautions and Warnings

All product profiles that run on top of OSGi runtimes are stopped by
the IBM webMethods Update Manager and will automatically be started
after the session completes.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes
for the release.

SIN-1769
In IBM webMethods Security Infrastructure, a security issue with password-protected files
in the third-party library Apache Velocity was discovered.
The issue is now resolved.

SIN-1744
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities CVE-2024-22257.
This issue is resolved. Spring Security is upgraded to the stable
version 5.8.16.

SIN-1731
In some rare cases SAML static initialization gets bypassed, because of a race condition.
Issue is fixed

SIN-1725
Software AG Security Infrastructure uses the third-party engine
Apache velocity 2.3.0, which has security vulnerabilities.
The issue is resolved. Software AG Security Infrastructure now uses
Apache Velocity 2.4.0.

OGI-4639
Software AG Security Infrastructure uses the third-party Bouncy Castle
library v. 1.76, which has known vulnerabilities.
This issue is resolved. Security Infrastructure now uses the stable version
Bouncy Castle 1.78.1

SIN-1629
In Software AG Security Infrastructure, you cannot enable Web SSO to use SHA-256 signatures and digests for sending requests to IdPs.
This issue is resolved. To enable Web SSO to use SHA-256, modify the system property com.softwareag.security.opensaml.default.digest=http://www.w3.org/2001/04/xmlenc##sha256 or the environment variable OPENSAML_DEFAULT_DIGEST=http://www.w3.org/2001/04/xmlenc#sha256. To modify default signature algorithm you can specify it the following way com.softwareag.security.opensaml.default.sign=RSA_http://www.w3.org/2001/04/xmldsig-more##rsa-sha256 or OPENSAML_DEFAULT_SIGN=RSA_http://www.w3.org/2001/04/xmldsig-more#rsa-sha256.

SIN-1641
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities CVE-2024-22257.
This issue is resolved. Spring Security is upgraded to the stable
version 5.8.11.

OGI-3899
The Software AG Security Infrastructure uses ESAPI 2.4.0.0 that has known vulnerabilities.
This issue is resolved. ESAPI is upgraded to the stable version 2.5.2.0.

OGI-3878
Software AG Security Infrastructure uses the third-party Bouncy Castle library v. 1.69.0.04,
which has known vulnerabilities.
This issue is resolved. Security Infrastructure now uses the stable version Bouncy Castle 1.76.

OGI-3850
Software AG Security Infrastructure uses the third-party Apache Velocity Engine v.1.5,
which has a known vulnerability CVE-2020-13936.
This issue is resolved. Security Infrastructure now uses the stable version
Apache Velocity 2.3.

OGI-3806
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities CVE-2023-34035.
This issue is resolved. Spring Security is upgraded to the stable
version 5.8.6.

OGI-3484
The Software AG Security Infrastructure uses a version of
Spring Security that has known vulnerabilities.
This issue is resolved. Spring Security is upgraded to the stable
version 5.7.5.
