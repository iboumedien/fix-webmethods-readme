1.0 Fix Name

webMethods Monitor 10.5 MWS Fix 6 (Monitor portlets update)



2.0 Fix ID

MON_10.5_MWS_Fix6



3.0 Product(s)/Component(s) Affected

This fix affects webMethods Monitor 10.5.



4.0 Requirements and Dependencies

Monitor 10.5 Fix 6 or higher.  (WmMonitor package update)

The Optimize fixes contain updates to shared components used by both Monitor and Optimize products, 
so would be installed even if Optimize product not present on system.

Note:All optimize 10.5.0 Fix 7 or higher fixes should be also installed even if Optimize product not present on system

It is also recommended to update to latest fixes for related BPM products
such as Designer, Process Engine, Optimize and ESB, and Business Console

when the window is displayed.

The issue is resolved.

WMN-7151(MON_10.5_MWS_Fix3)
When searching in a custom process instance portlet, webMethods Monitor
can now display the search results in the same way that the search results
are displayed for the default process instance portlet.

To change how the search results are displayed for a custom process instance portlet:
1/ Install the fix.
2/Log in as sysadmin.
3/ Open the custom process instance page (Folders > My webMethods Applications >
Fabric Tasks > Monitoring > {Custom_Process_Name} > Search).
4/ Open the Properties page by clicking the arrow button of the Search control and change
the value of the searchResultBar property in the Portlet preferences panel to a variable
name (for example, "results.test"), instead of "webm.apps.mon.biz.process.instance.list.results".
5/ Open the custom process instance page (Folders > My webMethods Applications >
Fabric Tasks > Monitoring > {Custom_Process_Name} > Process Instances)
6/ Open the Properties page by clicking the arrow button of the Process Instances control and
go to the Maintenance section and add the variable that was set for
the searchResultBar property ("results.test" in the example in step 4) as an alias.
7/Log in as an administrator.

This issue is resolved.

Usage Notes:
- Use a unique alias for each custom process instance.

8.0 Installation

Use Command Central to install fixes on multiple machines from a central  location (9.7 or later

9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action
