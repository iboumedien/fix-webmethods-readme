INM-25626 (Fix 30)
The application does not validate for File Type or Content-Type in the avatar
field

Server-side validation is added for the file upload functionality in CentraSite.
The file upload fails if the file size is above the set limit(default 100 MB)
or the file is a binary (with mime type "application/octet-stream").

The maximum file size limit can be set in one of the config.xml files:
<CentraSiteInstall_Directory>/cast/cswebapps/BusinessUI/custom/conf/
centrasite.xml
<CentraSiteInstall_Directory>/cast/cswebapps/BusinessUI/system/conf/
centrasite.xml

Add the <FileUploadSettings maxUploadfileSize="" /> inside the UIProperties
tag.

For example:

<GUIConfiguration>
    ...
    <UIProperties>
        ...

        <FileUploadSettings maxUploadfileSize="104857600" />

        ...
    </UIProperties>
    ...
</GUIConfiguration>

The issue is resolved.

INM-25493 (Fix 27)
Change Policy from Load Balancer Routing to ST Rounting

Be sure to create a database backup before executing this script.

The tool consists of an executable jar file in the bin folder of the
CentraSite installation. It can be invoked using the CentraSiteToolbox
script located in <SAG_Install_Dir>\CentraSite\utilities in the following
manner:

Usage:
CentraSiteToolbox.[cmd|sh] CentraSiteVSRoutingAdapter.jar
            <arg1_CentraSite_DB_URL>
            <arg2_Administrator_User>
            <arg3_Administrator_User_Password>
            <arg4_UUID_of_asset_to_be_Change_policy>
            <arg5_Straight_Through_Routing_URL>

arg1_CentraSite_DB_URL - (Optional) The URL to connect to CentraSite.
                                      It should be of the form
                                      http://hostname:port/CentraSite/CentraSite
                                      Default value (if not provided)
                                      http://localhost:53307/CentraSite/CentraSite
arg2_Administrator_User - The username of the user having the
                                      CentraSite Administrator role
arg3_Administrator_User_Password - The password for the Administrator user
arg4_UUID_of_asset_to_be_Change_policy - The UUID of Virtual Service\Virtual REST Service
arg5_Straight_Through_Routing_URL - The URL to be used for Straight Through Routing.

Example:
CentraSiteToolbox.[cmd|sh] CentraSiteVSRoutingAdapter.jar
"http://hostname:53307/CentraSite/CentraSite" "DOMAIN\admin"
"p@s$w0rd" "uddi:3fb8dbd0-6122-11e5-933f-e8830d260cc1" "http://www.acme.com"

Runtime Alias - Linux OS

CentraSiteToolbox.[cmd|sh] CentraSiteVSRoutingAdapter.jar
"http://hostname:53307/CentraSite/CentraSite" "DOMAIN\admin"
"p@s$w0rd" "uddi:3fb8dbd0-6122-11e5-933f-e8830d260cc1" "\${ESB_DEV}/acme.wsdl"

Runtime Alias - Windows OS

CentraSiteToolbox.[cmd|sh] CentraSiteVSRoutingAdapter.jar
"http://hostname:53307/CentraSite/CentraSite" "DOMAIN\admin"
"p@s$w0rd" "uddi:3fb8dbd0-6122-11e5-933f-e8830d260cc1" "${ESB_DEV}/acme.wsdl"

Note : After running this tool, Changes will be reflected after restarting the
Software AG Runtime 10.5


INM-25584 (Fix 26)
Vulnerable third party component Log4j is used

Severe vulnerabilities affecting multiple versions (2.0 to 2.17.0) of the Apache
Log4j 2 utility have been publicly disclosed. These vulnerabilities allow
unauthenticated remote code execution.

execution.

INM-25280 (Fix 18)
Unable to modify or delete a design time policy

When a design time policy applicable to a policy itself is activated, it cannot
be deactivated. Consider a design time policy with the scope of Policy type and
Pre-Update event type and has a policy action that can fail the update of the
policy itself when executed. After such a policy is activated, it cannot be
deactivated since the policy itself fails the update or deactivation.

The issue is resolved.

INM-24487 (Fix2)
Legacy API Portal Unpublish action is now available

CentraSite 10.5 allows you to publish services to API Portal, which were already
published to Mediator. Now you can also unpublish services that are deployed  to
API Portal. To activate this feature, perform the following steps:
1. Open the centrasite.xml file from [INSTALL_DIR]/CentraSite/cast/cswebapps/BusinessUI/system/conf
2. Add the following line after the line containing string <Action id="LegacyPortalPublishAction":
   <Action id="LegacyPortalUnpublishAction" inbox="false" bulk="true" group="runtime" icon="images/system/actionbar/legacy_unpublish_32X32.png" class="com.softwareag.centrasite.api.csom.action.runtime.impl.LegacyPortalUnpublishActionImpl">INMCL_ACTION_LEGACY_PORTAL_UNPUBLISH</Action>
3. Save the configuration file and exit
4. Restart Software AG Runtime 10.5.

8.0 Installation

Software AG strongly recommends you to take a backup of the internal database in
which CentraSite's registry and repository data is stored before installing this
fix.

Install the fix using  Software AG Update Manager.  For  instructions,  see  the
documentation at http://documentation.softwareag.com.


8.1 Post-Installation Steps

It may be required to  start  Software AG  Runtime  and/or  CentraSite  Registry
Repository after fix installation.

Clear browser cache before starting CentraSite Control.

You cannot uninstall the fixes applied on CentraSite 10.5.



10.0 Globalization

This  fix  conforms  to the  internationalization  standards  of the  webMethods
product suite  and includes  support for  operation in  any  country, locale, or
language as specified  in the Installing  webMethods  Products guide. It was not
tested with non-English configurations and non-ASCII data. However, this fix has

locale or configuration.
