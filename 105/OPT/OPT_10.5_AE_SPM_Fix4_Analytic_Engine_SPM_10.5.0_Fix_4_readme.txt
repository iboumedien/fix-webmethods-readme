1.0 Fix Name

webMethods Analytic Engine SPM 10.5.0 Fix 4



2.0 Fix ID

OPT_10.5_AE_SPM_Fix4


3.0 Product(s)/Component(s) Affected

This fix affects:

- Platform Manager 10.5
- Optimize Analytic Engine SPM 10.5


4.0 Requirements

For language pack requirements, see the Globalization section
of the Software AG Empower Product Support website at
https://empower.softwareag.com/Products/Globalization/default.asp

For Optimize environments, the fix version of the Optimize component fixes must
be the same as the fix version of Optimize shared components fixes
(wMFix.OPT.optimizeSharedLib and wMFix.OPT.optimizeSharedUI).


5.0 Platform Support

Same platforms as product release.


6.0 Cautions and Warnings

1. Attempting to apply the Optimize Fix to an installation where
 MWS is configured to use the embedded Derby database will fail.
 There will be an error reported in the Software AG Update Manager
 log. For all other supported Relational Database Management Systems
 configured for MWS, the fix can be applied successfully.

 There is no workaround at this time.


 2. You cannot uninstall this fix using Software AG Update Manager.

 To be able to restore your Optimize installation, before you install
 the fix you must back up the Optimize database and your file system.


7.0 Fix Contents


OBE-11603 (OPT_10.5_AE_SPM_Fix4)
Optimize saves the text value of the SNMP Alert password in a configuration file. The issue 
occurs when you use Command Central to configure the the SNMP Alert settings for Analytic Engine.

The issue is resolved.




OBE-11548 (OPT_10.5_AE_SPM_Fix3)
An error might occur when you configure Optimize through Command Central.

When you configure Optimize using Command Central, the following exception might occur:
"ArrayIndexOutOfBoundsException".
The issue occurs when the database pool URL you set does not contain a port number.

The issue is resolved. 

OBE-11611 (OPT_10.5_AE_SPM_Fix3)
With Optimize Analytic Engine, using user passwords that are encrypted in Command Central fails.

In Command Central, on the Databases and JNDI configuration type pages for Analytic Engine, when you 
attempt setting a user password with an encrypted value, Optimize incorrectly considers it a 
password with a clear-text value. As a result, you cannot set an encrypted password.

The issue is resolved.


OBE-11574 (OPT_10.5_AE_SPM_Fix3)
In Command Central, you cannot edit the SNMPAlert Settings and the WSAction Settings for Analytic 
Engine. 

In Command Central, when you edit the SNMPAlert Settings or the WSAction Settings for Optimize 
Analytic Engine, the changes are saved in the wrong location. As a result, Analytic Engine continues 
to use the old settings.

The issue is resolved.



OBE-11521 (OPT_10.5_AE_SPM_Fix2)
You cannot view the password of an Optimize database pool in Command Central.

In Command Central, the Databases configuration type page for Optimize Analytic Engine might not 
display the existing password of a database pool. The issue occurs when the database pool is 
configured using the Central Configurator in My webMethods.

The issue is resolved.

OBE-11420 (OPT_10.5_AE_SPM_Fix2)
You might not be able to view existing Optimize database pools using Command 
Central.

Command Central might fail to display an existing Optimize database pool. The 
issue occurs when you use Command Central to update a database pool that is 
configured using the Central Configurator in My webMethods.

The issue is resolved.




OBE-11369 (OPT_10.5_AE_SPM_Fix1)
On the Databases configuration page for Analytic Engine in Command Central, in 
the RDBMS drop-down list you can now select a PostgreSQL database for Optimize 
Analytic Engine.

OBE-11286 (OPT_10.5_AE_SPM_Fix1)
When Optimize is configured with SSL through the Central Configurator, Platform 
Manager always reports that the status of Analytic Engine is STOPPED. 

The issue is resolved.

10.0 Globalization 

This fix conforms to the internationalization standards of the
webMethods product suite and includes support for operation in
any country, locale, or language as specified in the Installing
webMethods Products guide. It was not tested with non-English
configurations and non-ASCII data. However, this fix has no
globalization impact and can be applied to systems running in
any supported locale or configuration.
