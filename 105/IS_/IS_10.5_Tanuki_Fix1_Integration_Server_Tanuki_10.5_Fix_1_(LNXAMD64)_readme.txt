1.0 Fix Name

WebMethods Integration Server 10.5 Tanuki  Fix 1 readme



2.0 Fix ID

IS_10.5_Tanuki_Fix1



3.0 Product(s)/Component(s) Affected

This fix affects the common extended libraries installed with
the 10.5 webMethods products.

This fix upgrades the Tanuki Service Wrapper libraries to version 3.5.43.



4.0 Requirements and Dependencies

TPS_10.5_Tanuki_Fix1

Fixes are cumulative. Therefore, for each of the listed fixes, you can install the specified fix or 
later.

For language pack requirements, see the Globalization section of the Software AG Empower Product 
Support website at https://empower.softwareag.com/Products/Globalization/default.asp

Same platforms as product release.



6.0 Cautions and Warnings

None.


7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

PIE-67823
Integration Server does not restart after the shutdown and restart option is selected.

When the Shutdown and Restart link is selected in Integration Server Administrator and 
the Restart button is clicked, Integration Server may shut down but not restart. 
This issue is the result of the configuration files for the Java Service Wrapper 
not being updated to reflect the new version of the Tanuki jar file.

This issue is now resolved.



8.0 Installation

After installing this fix, perform the following steps to complete 
the changes

For each instance of Integration Server repeat steps 1 to 11

1. Shut down Integration Server.

2. Back up the following file:

   <SAG_install_dir>/profiles/IS_<instance>/bin/sagis103

3. Open sagis103 in a text editor.

4. Search all the occurrences of 3.5.X and replace text with 3.5.43

5. Save your changes and close the text editor.

6. Back up the following file:

<SAG_install_dir>/profiles/IS_<instance>/configuration/wrapper.conf

7. Open wrapper.conf in a text editor.

8. Remove the following text:
   wrapper.java.classpath.1=
   <SAG_install_dir>/common/lib/tw-3.5.X/wrapper.jar
   
   Add the following text:
   wrapper.java.classpath.1=
   <SAG_install_dir>/common/lib/tw-3.5.43/wrapper.jar

9. Remove the following text:
   wrapper.java.library.path.1=
   <SAG_install_dir>/common/lib/tw-3.5.X
   
   Add the following text:
   wrapper.java.library.path.1=
   <SAG_install_dir>/common/lib/tw-3.5.43
   
10.Save your changes and close the text editor.

11.Start Integration Server.

Install using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.



9.0 Uninstallation

Uninstall the fix using the Software AG Update Manager.
For instructions, see Using the Software AG Update Manager.

After uninstalling the fix, perform the following steps to restore
the wrapper.conf to the state it was in prior to making changes

For each instance of Integration Server repeat steps 1 to 3

1. Shut down Integration Server.

2. Replace the following files with the backup created while
   installing the fix:

<SAG_install_dir>/profiles/IS_<instance>/bin/sagis103
<SAG_install_dir>/profiles/IS_<instance>/configuration/wrapper.conf

3. Start Integration Server.

Note: These instructions can only be used to uninstall the most recently installed fix. This action 
will revert your installation to the previously installed fix. You cannot use these instructions to 
uninstall the previously installed fix.

Uninstall using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.
