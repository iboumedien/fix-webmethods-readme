7.0 Fix Contents

In addition to including resolutions to product defects, a fix may
also include enhancements to existing functionality and features.
Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully
before applying the fix.

CEL-417 (CEL_10.5_Fix6)
The elasticsearch client jar presents in the API Gateway contains vulnerable jars.

CEL-395 (CEL_10.5_Fix6)
Vulnerable third-party component, HttpClient, used.

This issue is resolved. The versions of the vulnerable jar files are upgraded.

CEL-368 (CEL_10.5_Fix5)

Security vulnerability CVE-2021-45105 reported on log4j 2.16.0

The issue is resolved. All Log4j libraries are updated to version 2.17.1.

In case searchgaurd plugin shipped in the Internal Data Store is installed in your instance,
We recommend to upgrade the log4j-slf4j-impl library to the safest version manually.

CEL-335 (CEL_10.5_Fix4)

Identified third party Log4j 2.9.1, which have known security vulnerability CVE-2021-44228

The issue is resolved. All Log4j libraries are updated to version 2.16.0.

In case searchgaurd plugin shipped in the Internal Data Store is installed in your instance,
We recommend to upgrade the log4j-slf4j-impl library to the safest version manually.


CEL-203 (CEL_10.5_Fix3)
When Internal Data Store 10.5 Fix2 is installed, it overrides InternalDataStore
/config/jvm.options file.

From fix 3, the default heap size is set to 1GB.
Please update the heap size in jvm.options as per the needs. 

CEL-192 (CEL_10.5_Fix3)
For the first time, CEL doesn't show the Elasticsearch clustering details properly.

CEL doesn't show the proper node names under the clustering configuration.

This issue is resolved now. 

CEL-119 (CEL_10.5_Fix2)
Incorrect status displayed in Command Central.

Though the Internal Data Store is running as a Windows server, it's status
is displayed as 'Stopped' in Command Central.

This issue is resolved.

CEL-116 (CEL_10.5_Fix1)
Default JVM heap size enhanced for Elasticsearch.

The default JVM heap size for Elasticsearch is increased from 1 GB to 2 GB.
The existing configurations -Xms1g and -Xmx1g are changed to -Xms2g and -Xmx2g 
respectively where:
-Xms2g represents the initial size of total heap space.
-Xmx2g represents the maximum size of total heap space.
