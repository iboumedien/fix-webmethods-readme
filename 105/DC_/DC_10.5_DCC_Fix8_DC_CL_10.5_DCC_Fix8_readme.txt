1.0 Fix Name

webMethods Database Component Configurator 10.5 Fix 8.



2.0 Fix ID

DC_10.5_DCC_Fix8



3.0 Product(s)/Component(s) Affected

This fix affects the webMethods Database Component Configurator of webMethods Database
Configuration.



4.0 Requirements

None.



5.0 Platform/RDBMS Support

This fix is supported on all platforms supported by webMethods Product Suite 10.5



6.0 Cautions and Warnings

None.



7.0 Fix Contents

WDA-1799
Not able to start dbConfiguratorUI after applying the DC_10.5_DCC_Fix7.

This issue is now resolved.

WDA-1712(DC_10.5_DCC_Fix7)
Due to database catalog parsing error in 10.5, Upgrade using Stacks is failing 
with the following error:
"SPMCOME0007: HTTP 500 Internal Server Error"

This issue is now resolved.  

WDA-1783(DC_10.5_DCC_Fix7)
Unable to start DCC UI due to NPE in javax.swing package.
This issue is now resolved. 


WDA-1608(DC_10.5_DCC_Fix6)	
If the --version option was omitted when running the DCC command-line application, 
the result would be an error.

This issue has been resolved. 

WDA-1675(DC_10.5_DCC_Fix6)	
The results area of the DCC GUI was rendered with a proportional font, reducing its legibility.

This issue has been resolved.

WDA-1651(DC_10.5_DCC_Fix6)	
When executing commands, the output in the results area and console was difficult to read, 
because of a lack of formatting.

This issue has been resolved.

WDA-1654(DC_10.5_DCC_Fix6)	
Updated MySQL storage scripts to support MySQL 8.0

WDA-1606(DC_10.5_DCC_Fix6)	
DCC component tracking had errors with MySQL 8.

This issue has been resolved.

WDA-1605(DC_10.5_DCC_Fix5)
DCC required the options --user and --password, even when using alternative 
security.

This issue has been resolved.


WDA-1608(DC_10.5_DCC_Fix5)
If the --version option was omitted when running the DCC command-line application,
the result would be an error.

This issue has been resolved.


WDA-1625(DC_10.5_DCC_Fix5)
If logLevel is set to DEBUG, DCC did not display the SQL statements being executed.

This issue has been resolved.


WDA-1596(DC_10.5_DCC_Fix5)
The DCC GUI did not render within the available space on some resolutions.

This issue has been resolved.


WDA-1550 (DC_10.5_DCC_Fix4)
It was possible to drop a database component that had a dependency from an 
installed component.

This issue has been fixed.


WDA-1537 (DC_10.5_DCC_Fix4)
The database configuration options "--import" and "--export" were not functional.

This issue has been fixed.


WDA-1571 (DC_10.5_DCC_Fix4)
The catalog action could show multiple versions of the same component as installed.

This issue has been fixed. 


WDA-1472 (DC_10.5_DCC_Fix3)
On error of the drop action, the exit status will be zero, indicating success.

This issue is resolved.


WDA-1533 (DC_10.5_DCC_Fix3)
Password can be written to the log on error.

This issue is resolved.


WDA-1538 (DC_10.5_DCC_Fix3)
Error reading from configuration file with backslashes.

This issue is resolved.


WDA-1542 (DC_10.5_DCC_Fix3)
Error processing an upper-case action parameter.

This issue is resolved.


WDA-1467 (DC_10.5_DCC_Fix2)
Error installing Optimize with PostgreSQL.

This issue is resolved.


WDA-1469 (DC_10.5_DCC_Fix2)
The DCC 10.5 GUI does not match the behavior of 10.3.

This issue is resolved.


WDA-1514 (DC_10.5_DCC_Fix2)
Error starting DCC when installed in a directory with space in the path.

This issue is resolved.


WDA-1533 (DC_10.5_DCC_Fix2)
User credentials are written to the log file if an error occurs when creating
the storage component.

This issue is resolved.


WDA-1484 (DC_10.5_DCC_Fix1)
Fixed issues with dbConfiguratorUI.bat as DCC UI not working.



8.0 Installation

Install the fix using the Software AG Update Manager. For instructions, see 
Using the Software AG Update Manager located either in the _documentation 
directory or on the documentation Web site at http://documentation.softwareag.com.



9.0 Uninstallation

Uninstall the fix using the Software AG Update Manager. For instructions, see 
Using the Software AG Update Manager.

NOTE: This uninstall procedure can only be used to uninstall the most recently 
installed fix. This action will revert your installation to the previously 
installed fix. You cannot apply this uninstall procedure to the previously 
installed fix.


 
9.1 Localization

This fix does not require an updated Language Pack. It might 
contain new messages and these messages will appear in English.
