1.0 Fix Name

Software AG Security Infrastructure 10.5 Fix 12



2.0 Fix ID

SIN_10.5_Fix12


3.0 Product(s)/Component(s) Affected


This fix affects SAG Security Infrastructure 10.5.



4.0 Requirements

This fix requires Software AG Security Infrastructure 10.5. 
Software AG Security Infrastructure 10.5 Fix 12 
must be installed together with OSGI.Platform_10.5.0_Fix1,wMFix.TPS.SpringF 
if the products for these fixes are installed on the system. 

Official fixes are cumulative. Therefore, you can install the specified fix or later.

For language pack requirements, see the Globalization section of the Software AG
Empower Product Support website at
https://empower.softwareag.com/Products/Globalization/default.asp



5.0 Platform Support

This fix is supported on all platforms supported
by webMethods Product Suite 10.5.



6.0 Cautions and Warnings

Software AG Tomcat Server is stopped by Software AG Update 
Manager, therefore it must be restarted after applying the fix.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before applying the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

SIN-1493
The Software AG Security Infrastructure uses a version of
ESAPI that has known vulnerabilities.
This issue is resolved. ESAPI is upgraded to the stable
version 2.4.0.0. 

SIN-1474
Security Infrastructure uses the third-party Spring-security library which has а security vulnerability.
This issue is resolved.

SIN-1413
In some cases keystore and truststore information for LDAPS connection was not properly loaded and secure connection could fail.

Issue is now resolved

SIN-1432
Upgrade of Spring Security to 5.5.2 for security reasons.

SIN-1374
When checking password of LDAP user when using technical user connection to LDAP server, resources of the connection could be left to be released later.
Now all resources are released immediately.

The issue is now resolved.

SIN-1391
If username contains LDAP special symbol and affixes are used authentication to LDAP might fail due to missing escape of the username.

The issue is now resolved.
 

SIN-1413
In some cases keystore and truststore information for LDAPS connection was not properly loaded and secure connection could fail.

Issue is now resolved

SIN-1374
When checking password of LDAP user when using technical user connection to LDAP server, resources of the connection could be left to be released later.
Now all resources are released immediately.

The issue is now resolved.

SIN-1391
If username contains LDAP special symbol and affixes are used authentication to LDAP might fail due to missing escape of the username.

The issue is now resolved.
 

SIN-1380
Added property to enable LDAP Login Module to log authentication errors at ERROR level
 

SIN-1380
Added property to enable LDAP Login Module to log authentication errors at ERROR level
 

SIN-1364
A security vulnerability exists in the third-party Spring Security library.

This issue is resolved.
 

SIN-1354
The Security Infrastructure fix is not applied correctly for SPM.

The issue is now fixed.
 

OGI-2604
A security vulnerability exists in the third-party Bouncy Castle library.
This issue is resolved. Software AG Common Platform now uses
Bouncy Castle 1.68, where this vulnerability is fixed.

SIN-1337
Assertion engine was not supporting general attributes in assertion, only string types

The issue is now resolved

SIN-1338
Assertion engine was not supporting signed response with embedded unsigned assertion, only signed assertions

The issue is now resolved
 

SIN-1323
Software AG Security Infrastructure generates high memory consumption.
This issue is resolved.
 

OGI-2069
A security vulnerability exists in the third-party Bouncy Castle library.

This issue is resolved.
 

SIN-1302
Fix log levels to avoid security vulnerability.
 

SIN-1289
Added option to internaluserrepotool to pass base64 encoded password.
Option in question is -b

SIN-1290
Now when LDAP server properties change, cached user connections are dropped and must be reconnected with the new properties



8.0 Installation


Use Command Central to install fixes on multiple machines from a central 
location (9.7 or later products), or Software AG Update Manager to install 
fixes on one machine at a time. For instructions, see the documentation at
http://documentation.softwareag.com.


9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently 
installed fix. This action will revert your installation to the previously 
installed fix. You cannot use these instructions to uninstall the previously 
installed fix.

Uninstall using Software AG Update Manager. For instructions, see the 
documentation at http://documentation.softwareag.com.
