7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include 
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it is an aggregate of previous fixes 
for the release.

PIEAR-2573
When CSRF is enabled, Integration Server Administrator > webMethods Adapter for
Apache Kafka > Listeners page fails to provide an option to delete a disabled
listener.

This issue is now resolved.

PIEAR-2536
After applying the Adapter Runtime 10.5 Fix 8, the webMethods adapter for JDBC 
could not reload environment variables in some instances. 

This issue is now resolved.

PIEAR-2535
After applying the Adapter Runtime 10.5 Fix 8, the Adapter Runtime was throwing 
a NullPointerException (NPE) during startup in a few instances. 

This issue is now resolved.

PIEAR-1672
The /health endpoint does not return information about the adapter health
indicator when the Microservices Runtime or a Integration Server instance is
migrated from 10.3 version to 10.7 version.

This issue is now resolved.

Note: The /health endpoint functionality is available in Microservices Runtime
or in an Integration Server instance with a Microservices license.

PIEAR-2447 (WAR_10.5_Fix8)
In Microservices Runtime Administrator, adapters are not listed due to incorrect  
compiler code optimization.

This issue is now resolved.

PIEAR-2415 (WAR_10.5_Fix8)
When the health end-point of an adapter is accessed, the listener properties
appear including the runtimeState attribute which is a dictionary of key/value
pairs. If the listener is enabled, then the runtimeState dictionary displays the
runtime state of the listener. If the listener is disabled, then the runtimeState
dictionary displays only currentState as disabled which is the state of the
listener set by the user.

PIEAR-1760 (WAR_10.5_Fix8)
In webMethods Adapter Runtime, the administrator API GET operations for
notifications do not display the notification type and the notification state.

This issue is now resolved.

PIEAR-2256 (WAR_10.5_Fix8)
Whenever Integration Server is started, a WmAS400 package containing a single
config sub-directory is created.

This issue is now resolved.

PIEAR-2314 (WAR_10.5_Fix8)
In the webMethods Adapter Runtime, an administrator API request to enable a
suspended polling notification fails.

This issue is now resolved.

PIEAR-2315 (WAR_10.5_Fix8)
In webMethods Adapter Runtime, the administrator API GET operations for
notification do not display the notificationProperties field if the notification
type is a listener. The notificationProperties field is displayed for polling
notifications.

PIEAR-2320 (WAR_10.5_Fix8)
In webMethods Adapter Runtime, the notification state displayed in the
administrator API GET operations are not in sync with the notification status
displayed in the Integration Server Administrator > Server > Scheduler page.

This issue is now resolved.

PIEAR-1949 (WAR_10.5_Fix7) 
When CSRF guard is enabled, the user is not able to enable or disable connections
from the Integration Server Administrator screen if opened using chrome or edge
browser.

This issue is now resolved. 

PIEAR-1899 (WAR_10.5_Fix7) 
In Microservices Runtime, the JDBC polling notifications are not enabled although
the notificationEnabled property is set to true in the application.properties file.

This issue is now resolved.

PIEAR-1889 (WAR_10.5_Fix7) 
The adapter asset properties passed as environment variables arguments in a
docker run command are not substituted for the externalized adapter asset
properties defined in the application.properties file.

This issue is now resolved. 

PIEAR-1888 (WAR_10.5_Fix7) 
The listeners in webMethods Adapter for Websphere MQ, which are in pending enabled
state fail to change to enabled state when Integration Server exits the quiesce
mode. 

This issue is now resolved. 

PIEAR-1886 (WAR_10.5_Fix7) 
In webMethods Adapter Runtime, the WmSAP adapters' listener notification nodes
are not listed in Integration Server Administrator screen.

This issue is now resolved.

PIEAR-1788 (WAR_10.5_Fix7) 
The webMethods AdapterRuntime (ART) based adapters configuration properties
which can be configured from Integration Server Administrator are available in
configuration variables template generated by Microservices Runtime.
For more information, see the adapter installation and user guide.

PIEAR-1733 (WAR_10.5_Fix7) 
When the adapter listeners package is loaded before the adapter package, the
listeners are in Pending Enabled state and fail to change to the Enabled state.

This issue is now resolved.

If the adapter listeners package is loaded before the adapter package, then the
listeners are in the Disabled state and you must enable the listeners manually.
You must ensure proper package dependency is maintained to avoid the error. 

PIEAR-1854 (WAR_10.5_Fix6)
In Integration Server Administrator screen, on selecting the Adapters dropdown,
the installed adapters are not listed.

This issue is now resolved. 

PIEAR-1784 (WAR_10.5_Fix6)
The webMethods AdapterRuntime (ART) asset properties which can be configured
from Integration Server Administrator are available in configuration variables
template generated by Microservices Runtime. Microservices Runtime generates an
application.properties file.

You can update the values of the properties in the application.properties file
and pass to a Microservices Runtime. Microservices Runtime loads the information
from the application.properties and updates the webMethods Adapter Runtime assets
configuration properties.

PIEAR-1409 (WAR_10.5_Fix5)

PIEAR-1515 (WAR_10.5_Fix4)
WebSphere MQ listeners are not displayed in Integration Server Administrator.
A runtime exception occurs while registering the listener nodes as the 
corresponding adapter is not loaded. The runtime exception abruptly ends the 
listener node registration in that package. 

The issue is resolved.

PIEAR-1493 (WAR_10.5_Fix4)
In webMethods WebSphere MQ Adapter, when the connection in the Integration 
Server Admin UI page is enabled, invoking the 
pub.art.connection:queryConnectionState service causes a delay in Put service.
Invoking the pub.art.connection:queryConnectionState service creates a new 
connection to check the connectivity causing the delay.

The issue is now resolved.

A new watt property 
"watt.art.pub.queryConnectionState.disableConnectionCreation" is introduced to 
disable creating the connection in the pub.art.connection:queryConnectionState 
service. 
Set the property "watt.art.pub.queryConnectionState.disableConnectionCreation" 
to true to disable creating the connection. 
The pub.art.connection:queryConnectionState service returns the current state 
of the connection.

PIEAR-1467 (WAR_10.5_Fix4)
Unable to shutdown Integration Server because of a deadlock in webMethods Adapter Runtime.
The deadlock occurs at times when connection and listener cleanups happen concurrently.

The issue is resolved.

PIEAR-1463 (WAR_10.5_Fix4)

PIEAR-1441 (WAR_10.5_Fix3)

PIEAR-1444 (WAR_10.5_Fix3)
