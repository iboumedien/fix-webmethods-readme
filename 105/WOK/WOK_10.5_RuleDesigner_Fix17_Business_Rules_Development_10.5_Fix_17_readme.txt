7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in 
services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes 
for the release.

WOK-14116 (WOK_10.5_RuleDesigner_Fix3)
No error visible when rule project assembly fails
If an error occurs during the assembly of a rule project that contains Action definitions in either 
Asset Build Environment, Designer or the Business Rules UI in My webMethods Server, the error will 
not be visible to the user.
This issue has been resolved.

WOK-15328 (WOK_10.5_RuleDesigner_Fix11)
Cloud Deployment support
The Business Rules Development perspective in Designer now supports "Deploy to Cloud" to a Cloud 
Container solution.
This issue has been resolved.

WOK-15450 (WOK_10.5_RuleDesigner_Fix14)
DeployToCloudException: Could not promote assets when deploying rule project to Cloud Container
When an IS package has been previously deployed to a Cloud Container solution and then a rule 
project is deployed to the same solution, the rule project deployment fails with an 
"DeployToCloudException: Could not promote assets when deploying rule project to Cloud Container" 
error.
This issue has been resolved.

WOK-15528 (WOK_10.5_RuleDesigner_Fix13)
Deployment of business rules fails with java.lang.ClassNotFoundException: 
com.softwareag.rules.runtime.pool.EnginePoolStore$Lambda$172
When deploying a business rules project to one or more Integration Servers, through Hot Deployment 
from My WebMethods Server, or through Deployer, sometimes the deployment fails on one or more 
Integration Servers. The Integration Server error log shows a java.lang.ClassNotFoundException: 
com.softwareag.rules.runtime.pool.EnginePoolStore$Lambda$172.
This issue has been resolved by replacing the sometimes problematic lambda expression with a 
concrete anonymous class.

WOK-15663 (WOK_10.5_RuleDesigner_Fix15)
NullPointerException during rule project builds
When building a rule project with the Asset Build Environment, the build would fail with
java.lang.NullPointerException
at com.wm.lang.ns.NSRecordRef.getNSRecord(NSRecordRef.java:245)
This issue is now resolved.

WOK-15728 (WOK_10.5_RuleDesigner_Fix14)
Cloud Container solutions list is not updated while deploying rule project
When beginning to deploy a rule project to cloud container using Designer, the list of available 
target solutions does not show recently created solutions.
This issue has been resolved.

WOK-15732 (WOK_10.5_RuleDesigner_Fix14)
No Designer error message on Deploy to a solution with a different version
When deploying rule projects from a 10.5 Designer installation to a 10.7 cloud container solution, 
the deployment fails with no error message and no user feedback.
This issue has been resolved by providing an error message, informing the user that the Designer 
installation and the cloud container solution must be on the same version.

WOK-15742 (WOK_10.5_RuleDesigner_Fix14)
Rules can be deployed from Designer to cloud container solution without the WmBusinessRules package
When deploying a rule project from Designer to a cloud container solution, the rule project can be 
deployed, even though the target solution does not have the WmBusinessRules package enabled.
This issue has been resolved.

WOK-15743 (WOK_10.5_RuleDesigner_Fix14)
Rules deployment is not asking to select IS for type3 solutions in Designer
When deploying a rule project from Designer to a cloud container solution which contains more than 
one Integration Server, the Designer Deploy to Cloud wizard does not prompt the user to select an 
Integration Server to deploy to.
This issue has been resolved.

WOK-15751 (WOK_10.5_RuleDesigner_Fix14)
Rules deployment is not tracking in history of Deploy Page
When deploying a rule project from Designer to a cloud container solution, the cloud container 
Solutions - Deploy - History page does not show information about the deployment.
This issue has been resolved.

WOK-15887 (WOK_10.5_RuleDesigner_Fix14)
10.5 Designer can't deploy to 10.5 solution
When using 10.5 Designer to deploy a rule project to a 10.5 Cloud Container solution, the 
deployment will fail with a NoClassDefFoundError: org/apache/commons/io/FileNameUtils
This issues has been resolved.

WOK-16172 (WOK_10.5_RuleDesigner_Fix16)
Missing dependency checks when deploying to Cloud Container
When using Designer to deploy rule projects to Cloud Container,
dependencies to External Event Rules, Process Actions and Predictive Analytics Actions were not 
checked and deployment of such rule projects was not prevented; dependencies to service actions 
were not checked and user was not informed of missing dependencies; dependencies to internal rules 
assets were declared.
This issue has been resolved.

WOK-16333
Jackson library v2.12.3 causes com.fasterxml.jackson.databind.exc.InvalidDefinitionException
When using Jackson library version 2.12.3, reading Business Rules auditing information from the 
database results in a com.fasterxml.jackson.databind.exc.InvalidDefinitionException.
This issue has been resolved.
