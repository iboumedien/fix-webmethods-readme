1.0 Fix Name

My webMethods Server Database 10.7 Fix 4



2.0 Fix ID

MWS_10.7_Database_Fix4



3.0 Product(s)/Component(s) Affected

My webMethods Server 10.7, Task Engine 10.7



4.0 Requirements and Dependencies

Fixes are cumulative. Therefore, for each of the listed fixes, you can install the specified fix or 
later.

For language pack requirements, see the Globalization section of the Software AG Empower Product 
Support website at https://empower.softwareag.com/Products/Globalization/default.asp



5.0 Platform/RDBMS Support

Same platforms as product release.



6.0 Cautions and Warnings

Uninstalling this fix does not revert the database component level for My webMethods Server.
To be able to uninstall the fix later, stop My webMethods Server and back up the My webMethods
Server database before installing the fix.


7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

WTN-5861 (MWS_10.7_Database_Fix4)
Some of the scripts that create Task Engine indexes are not included in the database scripts for MySQL.

The issue is resolved.

MWS-29368 (MWS_10.7_Database_Fix3)
The database scripts for My webMethods Server create some of the database indexes in the data
tablespace instead of the index tablespace.

The issue is resolved.


MWS-30078 (MWS_10.7_Database_Fix3)
When migrating the My webMethods Server database from version 10.7 or earlier, the Database
Component Configurator registers two additional components. For Oracle and DB2 databases, the
records added to the COMPONENT_EVENT table do not fill the tablespace-related columns, and might
return null when executing a Catalog operation.

The issue is resolved. The tablespace-related columns are now populated properly during migration.


MWS-30107 (MWS_10.7_Database_Fix3)
Attempting to re-execute the migration of the database component for My webMethods Server fails
with a "java.sql.SQLSyntaxErrorException: [SoftwareAG][SQLServer JDBC Driver][SQLServer]Column
names in each table must be unique. Column name 'SOURCE' in table 'TBLPORTALVERSION' is specified
more than once." error message. This issue occurs when upgrading from My webMethods Server
version 10.5 to version 10.7 with fix 1.

The issue is resolved.


MWS-30844 (MWS_10.7_Database_Fix3)
The Database Component Configurator might set an incorrect database level during My webMethods
Server upgrade.

When migrating the database component for My webMethods Server from a version older than 10.7, the
database scripts for My webMethods Server register two new database components with level 10.7,
regardless of the version you are upgrading to, and cause errors during subsequent runs.

The issue is resolved.


WTN-5418 (MWS_10.7_Database_Fix2)
Customer could not archive all tasks using Oracle database. Customer also had deleted task types
without deleting task instances for the particular deleted task type, which prevented tasks from
being archived.

The issues have been resolved.


MWS-26590 (MWS_10.7_Database_Fix1)
My webMethods Server version 9.10 does not provide migration scripts for database columns, related
to the UUID functionality which is enabled in a fix.

The issue is resolved. The migration scripts, available in My webMethods Server version 9.10 now
include scripts for UUID columns. The expected db component version when upgrading to My
webMethods Server 9.10 is now 47.1. For the fix to take effect, install the corresponding
required fix (see Requirements), and upgrade the database after the fix installation.


MWS-28568 (MWS_10.7_Database_Fix1)
The database migration scripts for My webMethods Server might fail. This issue occurs when
rerunning the Database Component Configurator, and the MWS_UPDATE_PKS (10.5) or MWS_UPDATE_TE
(10.7 or later) database procedures executed partially on a previous run.

The issue is resolved.


MWS-29236 (MWS_10.7_Database_Fix1)
When bootstrapping after upgrade, Common Directory Services in Integration Server might fail to
initialize completely and transfer all available settings from the source version.

The issue is resolved.




8.0 Installation

Install using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.

1. If this is the first time you are installing a fix using the Software AG Update Manager, stop 
   My webMethods Server and back up the My webMethods Server database and file system before 
   applying the fix.   

2. Important: See Section 6.0 Cautions and Warnings, point 6 in case you want to apply this fix to 
   an environment being upgraded.



9.0 Uninstallation

Although you can uninstall this fix using Software AG Update Manager, you cannot revert the
My webMethods Server DB level. Uninstalling this fix only removes the Database scripts for
My webMethods Server.

See Section 6.0 Cautions and Warnings for more information.

These instructions can only be used to uninstall the most recently installed fix. This action 
will revert your installation to the previously installed fix. You cannot use these instructions to 
uninstall the previously installed fix.

Uninstall using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.

You can only uninstall one My webMethods Server Database fix at a time. Make sure you backup your server 
settings before uninstalling My webMethods Server Database fixes.



10.0 Globalization

This fix conforms to the internationalization standards of the webMethods product suite and includes 
support for operation in any country, locale, or language as specified in the Installing webMethods 
Products guide. It was not tested with non-English configurations and non-ASCII data. However, this 
fix has no globalization impact and can be applied to systems running in any supported locale or 
configuration.



10.1 Localization

This fix does not require an updated language pack. It might contain new messages and these messages 
will appear in English.
