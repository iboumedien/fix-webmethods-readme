WAK-477
IBM webMethods Adapter for Apache Kafka provides a mechanism to handle the poison
pill scenario by leveraging an error-handling deserializer. The poison pill
scenario occurs when a record published to a Kafka topic continuously fails to
be consumed. This occurs when the producer and consumer are configured with
incompatible serializers and deserializers.

Perform the following steps to enable an error-handling deserializer:
1. Set watt.adapter.kafka.use.error.handling.deserializer property to true in
Integration Server Administrator, navigate to Settings > Extended.
2. Edit the relevant consumer connection to specify a custom service to process
the record with the error in the "Deserializer Error Handler" field
3. Enable the relevant consumer connection.

Note:
1. Consumer records with errors will not be passed to the consumer service or
configured listeners.
2. If the watt.adapter.kafka.use.error.handling.deserializer property is set to
true and the flow service is not specified in the "Deserializer Error Handler"
field, then the records with errors will be lost. The user may have to reset the
consumer offset to a prior value to read them again.
3. For more information about the input structure of the record with the error
received from the deserializer error handler, see the
wm.adapter.wmkafka.serde.specifications:deserializerErrorHandler sample service
specification.

for a message by subscribing or assigning topics.

committing of Kafka messages occurs.
