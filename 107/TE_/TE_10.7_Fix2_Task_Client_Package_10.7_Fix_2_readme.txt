7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes 
for the release.

WTN-5940 (TE_10.7_TC_Fix2)
Installing Task Client package fixes on Integration Server might overwrite the connection settings
to My WebMethods Server with the default values.

This issue is resolved.




MWS-27942 (TE_10.7_TC_Fix1)
The implementation of the environment properties processor for WmTaskClient is not optimized.

The issue is resolved.




WTN-5240 (TE_10.7_TC_Fix1)
When installing the Task Client package on Microservices Runtime, the required dependency to My
webMethods Server Libraries does not get selected automatically and all Task Engine services, or
attempts to modify the default configuration of the package fail with an error.

The issue is resolved. Now the My webMethods Server Libraries node under Infrastructure >
Libraries gets selected automatically when installing the Task Client package on Microservices
Runtime.




WTN-5663 (TE_10.7_TC_Fix1)
The formatting of the configuration page for the WmTaskClient package in Integration Server might
break after applying specific server settings.

After setting the watt.server.http.X-Content-Type-Options property for Integration Server to
nosniff, the Task Client configuration page loses the content formatting and alignment.

The issue is resolved.
