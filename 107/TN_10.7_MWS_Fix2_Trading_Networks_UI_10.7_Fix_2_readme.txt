7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services, APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes 
for the release.

OTN-22642
In webMethods Trading Networks, while creating a public search using the filter 
option, you select as a default search (options tab >Basic as a default search 
tab). Then, when you open the transaction page, it opens on the advanced tab, 
not the basic tab and switching to basic clean the content and return to â€œAll.â€?

This issue is now resolved. 

OTN-22955
The error message "Can't find resource for bundle
com.webmethods.portal.framework.portletcontainer.portletapi.impl.PortletResourceBundle,
key error.message.26022" appears on clicking the "Add Partner Group" button or
when editing an existing partner group in Partner group.

This issue is now resolved.

OTN-23038
When you submit a larger XML file on the submit document page, the Mywebmethods server fails and displays the error message "400: Unable to parse form content" 
in the UI.

This issue is resoved.

OTN-23138
In webMethods Trading Networks, after installing the Japanese language pack the
TN transactions "Query or Transform failed" and the partners are recognized as
'Unknown'

OTN-23017
When My webMethods receives the xml document with the CDATA section. Xml CDATA 
makes My webMethods unresponsive.
This issue is resolved

OTN-21604 (TN_10.7_MWS_Fix1)
In MWS, error message gets displayed in the attributes section of the transaction
when the submitted document did not have the StringList which was intended to be
extracted.
This issue is now resolved.

OTN-21816 (TN_10.7_MWS_Fix1)
webMethods IS is running on Atlantic time zone.
Site had setup to suspend delivery on March 13th 2021 from 11 PM to 12 PM EST.
Once the suspension is setup, it's displaying date as 2021/03/13 22:00 - 
2021/03/13 23:00 with time zone is setup as Easter time.
But the scheduler on IS are created correctly from March 14th 00 to 
March 14th 00 (AST to EST 1 hours difference).

OTN-21884 (TN_10.7_MWS_Fix1)
When a user logs into MWS with OAuth user, functionalities such as Submit
Documents, Configure Properties, Partner Group deletion, and Processing Rule
creation does not function properly. The user gets an HTTP error response on the screen.
This issue is resolved now.

OTN-21890 (TN_10.7_MWS_Fix1)
when you create any public queue and copy it using copy action and save it after
that whenever you try to create any new public queue it will always append
"Copy of" String in the newly created queue.

This issue is now resolved.
