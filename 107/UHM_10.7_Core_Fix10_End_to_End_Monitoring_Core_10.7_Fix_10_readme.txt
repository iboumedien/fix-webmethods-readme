6.0 Cautions and Warnings

Applying a fix reverts the changes in these files to their default values. "E2EMonitoring\agent\config\agent.config",
"E2EMonitoring\agent\config\apm-trace-include-plugin.config" and "E2EMonitoring\agent\e2ecustomlogback.xml".
Create a backup of these files before applying the fix, if you have made changes to these files. After the fix is 
applied, reapply the changes.



7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

UHM-4232 (UHM_10.7_Core_Fix10)

With End to End Monitoring hybrid configurations Integration server WmCloud tenant connection page was breaking.

This issue is now resolved.



UHM-3545 (UHM_10.7_Core_Fix8)

End to End Monitoring Hybrid agent to collector connection was not happening when 2 way SSL was enabled in Integration Server.

This issue is now resolved.



UHM-3295 (UHM_10.7_Core_Fix7)

With log4j usage in End to End monitoring Integration server logs used to contain improper field values.

This issue is now resolved, End to End Monitoring now uses logback for logging support.



UHM-3106 (UHM_10.7_Core_Fix6)

For Hybrid monitoring to work collector endpoint needed to be manually added in the agent.config.

Now this issue is resolved, Collector endpoint is dynamically fetched from the tenant connection details provided in 
Integration server.



UHM-2892 (UHM_10.7_Core_Fix5)

End to End Monitoring used to trace Integration server services for which audit needed to be enabled in Software AG Designer. 
For each service audit needed to be enabled explicitly.

This issue is now resolved, audit for services can be now enabled as part of Integration servers extended settings.



UHM-2644 (UHM_10.7_Core_Fix4)

Log4J Zero-Day vulnerability issue.

This issue is resolved. End to End monitoring now contains advanced measures.



UHM-2630 (UHM_10.7_Core_Fix3)

Log4J Zero-Day vulnerability issue.

This issue is resolved.



UHM-2546 (UHM_10.7_Core_Fix2)

End-to-End monitoring fails when Integration Server is enabled with multiple tenant connections.

This issue is resolved. End-to-End monitoring now successfully sends data to cloud for multiple 
tenant connections in the same region.



UHM-2380 (UHM_10.7_Core_Fix1)

Log4j properties file "e2ecustomlog4j2.properties" is required for hybrid integration monitoring functionality. 
This file does not load on Integration Server startup for hybrid integration.

This issue is resolved. All logs generated by agent pertaining to segment write, data transfer summary, 
and failed segment send to cloud are now written to log4j log files.



8.0 Installation

9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action will 
revert your installation to the previously installed fix. You cannot apply these instructions to the 
previously installed fix.

Uninstall using the Software AG Update Manager. For instructions, see the documentation 
at http://documentation.softwareag.com.

This fix has dependency on a java fix. Uninstalling java fixes does not revert changes made for
End-to-End Monitoring.
