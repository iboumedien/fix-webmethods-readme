7.0 Fix Contents

UHM-4232 (UHM_10.7_ISOrMSRPlugin_Fix10)

With End to End Monitoring hybrid configurations Integration server WmCloud tenant connection page was breaking.

This issue is now resolved.



UHM-3545 (UHM_10.7_ISOrMSRPlugin_Fix8)

End to End Monitoring Hybrid agent to collector connection was not happening when 2 way SSL was enabled in Integration Server.

This issue is now resolved.



UHM-3295 (UHM_10.7_ISOrMSRPlugin_Fix7)

With log4j usage in End to End monitoring Integration server logs used to contain improper field values.

This issue is now resolved, End to End Monitoring now uses logback for logging support.



UHM-3106 (UHM_10.7_ISOrMSRPlugin_Fix6)

For Hybrid monitoring to work collector endpoint needed to be manually added in the agent.config.

Now this issue is resolved, Collector endpoint is dynamically fetched from the tenant connection details provided in 
Integration server.



UHM-2892 (UHM_10.7_ISOrMSRPlugin_Fix5)

End to End Monitoring used to trace Integration server services for which audit needed to be enabled in Software AG Designer. 
For each service audit needed to be enabled explicitly.

This issue is now resolved, audit for services can be now enabled as part of Integration servers extended settings.



UHM-2547 (UHM_10.7_ISOrMSRPlugin_Fix2)

End-to-End monitoring fails when Integration Server is enabled with multiple tenant connections.

This issue is resolved. End-to-End monitoring now successfully sends data to cloud for multiple 
tenant connections in the same region.



UHM-2381 (UHM_10.7_ISOrMSRPlugin_Fix1)

Child service traces are not generated in End-to-End Monitoring, when nested services are invoked asynchronously from an Integration Server service.
This issue is resolved.



8.0 Installation

Install using Software AG Update Manager. For instructions, see the documentation at 
http://documentation.softwareag.com.

1. If this is the first time you are installing a fix using the Software AG Update Manager, 
   back up the End-to-End Monitoring file system before applying the fix.   

2. After fix installation, copy \E2EMonitoring\agent\plugins\uhm-onpremise-is-plugin.jar to
   \IntegrationServer\lib\jars directory.



9.0 Uninstallation

Note: These instructions can only be used to uninstall the most recently installed fix. This action will 
revert your installation to the previously installed fix. You cannot apply these instructions to the 
previously installed fix.

Uninstall using the Software AG Update Manager. For instructions, see the documentation 
at http://documentation.softwareag.com.

After fix uninstallation, To revert the Software AG product installation file system to the state before fix installation,
delete changes made to the Software AG file system, along with this jar file \IntegrationServer\lib\jars\uhm-onpremise-is-plugin.jar
