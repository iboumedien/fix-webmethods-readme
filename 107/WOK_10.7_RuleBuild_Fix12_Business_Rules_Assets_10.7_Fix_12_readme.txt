7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include enhancements to 
existing functionality and features. Any of these items can result in new or changed built-in 
services,
APIs, or configuration parameters. Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the release.

WOK-15663 (WOK_10.7_RuleBuild_Fix4)
NullPointerException during rule project builds
When building a rule project with the Asset Build Environment, the build would fail with
java.lang.NullPointerException
at com.wm.lang.ns.NSRecordRef.getNSRecord(NSRecordRef.java:245)
This issue is now resolved.

WOK-16172 (WOK_10.7_RuleBuild_Fix5)
Missing dependency checks when deploying to Cloud Container
When using Designer to deploy rule projects to Cloud Container,
dependencies to External Event Rules, Process Actions and Predictive Analytics Actions were not 
checked and deployment of such rule projects was not prevented; dependencies to service actions 
were not checked and user was not informed of missing dependencies; dependencies to internal rules 
assets were declared.
This issue has been resolved.

WOK-16252 (WOK_10.7_RuleBuild_Fix7)
Deploying rule projects deletes existing projects on Cloud Container
When using the Asset Build Environment to deploy Business Rules assets to Cloud Container 
solutions, previously deployed Business Rules assets are deleted and only the newly deployed 
Business Rules assets are available.
This issue is now resolved by adding a "RULES.skip.clean=false|true" parameter to the 
"<sag-installation>\common\AsseBuildEnviroment\master_build\build.properties" file.

WOK-17935
Vulnerable 3rd Party Component commons-text used
Business Rules used a vulnerable version of commons-text: CVE-2022-42889.
This issue has been resolved by upgrading commons-text from from v1.8.0 to v1.10.0.
